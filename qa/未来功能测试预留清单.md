## L1.1 版本（预计 2026-09）

### 功能：K 值调整优化

> **背景**：L1 K=1 可能导致减重维度区分度低（见测试策略第九节）。

| 测试项 | 说明 | 优先级 |
|---|---|---|
| K 值修改后所有积分用例重算 | TC-SCORE-001~015 全部重算手动预期值 | P0（届时） |
| 历史数据迁移验证 | K 值变更是否触发历史 PK 结算数据重算或仅影响新 PK | P0 |
| K 值调整对并列排名的影响 | 新 K 值下是否产生更多并列场景 | P1 |

### 功能：S3 补卡机制

> **背景**：L1 不做补卡（DR-002 Q6 建议），L1.1 视情况引入。

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 补卡次数限制 | 最多 2 次补卡，超过第 3 次拒绝 | P0 |
| 补卡时间窗口 | 只能补昨日/前日，还是任意历史日期 | P0 |
| 补卡积分计算 | 补卡是否计入打卡率分子，减重分如何取值 | P0 |
| 补卡防刷 | 连续补卡多天验证（不能跨越 2 次上限） | P1 |
| 并发补卡 | 同时提交两个补卡请求，只成功一次 | P1 |

### 功能：S4 发起人管理权

> **背景**：房主踢人、改名、提前结束 PK。

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 踢人功能 | 房主踢出成员，被踢者不参与后续结算 | P0 |
| 踢人后历史数据 | 被踢成员历史打卡数据保留方式 | P0 |
| 改名 | 房间改名后排行榜/战报展示同步 | P1 |
| 提前结束 PK | 触发提前结算，与正常结算逻辑一致 | P0 |
| 权限边界 | 普通成员无法使用房主管理权限 | P0 |

### 功能：打卡水印服务端校验（L2 迭代，但 L1.1 可提前评估）

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 服务端水印验证 | 前端 Canvas 水印 + 服务端二次校验防篡改 | P0（L2） |
| 旧图检测 | 通过 EXIF 时间戳检测是否为历史照片 | P1（L2） |
| 异常图片拦截 | 上传无水印或水印内容异常的图片被服务端拒绝 | P0（L2） |

### 功能：S2 进度看板

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 体重曲线图数据准确性 | 曲线数据点与实际打卡体重一致 | P0 |
| 剩余天数计算 | 当天与 PK 结束日的日期差正确 | P0 |
| 曲线图空数据状态 | 仅一个数据点或零数据点时图表展示 | P1 |
| 断打卡区间 | 有几天未打卡，曲线如何连线 | P1 |

## L2 版本（预计 2026-Q4）

### 功能：打卡提醒（订阅消息）

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 订阅消息授权流程 | 用户授权订阅消息推送 | P0 |
| 推送时间准确性 | 设定的提醒时间（如 21:00）准时推送 | P0 |
| 消息内容正确性 | 消息内容包含 PK 名称、今日打卡状态 | P0 |
| 已打卡不推送 | 当日已打卡则不推送提醒 | P0 |
| 取消订阅 | 用户取消后不再推送 | P1 |
| 大量用户并发推送 | 多用户同时触发推送，消息不丢失 | P1 |

### 功能：食物识别 / 饮食 AI 分析

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 食物图片识别准确性 | 常见食物识别率验证 | P0 |
| 识别失败兜底 | 无法识别时显示"手动输入"引导 | P0 |
| 热量计算准确性 | 识别结果热量估算与参考值对比 | P1 |
| 隐私合规 | 用户上传食物图片不保留在服务端 | P0 |

### 功能：好友 / 社交互动

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 好友点赞 | 对他人打卡点赞，计数准确 | P0 |
| 好友评论 | 评论字数限制、内容安全过滤 | P0 |
| 评论通知 | 收到评论时推送通知（需订阅授权） | P1 |
| 陌生人匹配 PK | 随机匹配场景下隐私保护级别 | P0 |

### 功能：报名费 / 押金

> **注意**：涉及资金，测试优先级最高，需要独立的测试计划。

| 测试项 | 说明 | 优先级 |
|---|---|---|
| 微信支付完整流程 | 发起支付→支付成功→加入 PK | P0 |
| 支付失败处理 | 支付失败不加入 PK，金额不扣除 | P0 |
| 退款逻辑 | 解散房间退款、中途退出退款条件 | P0 |
| 结算分成 | 按排名分配奖金，精度验证（分钱级别） | P0 |
| 对账 | 支付流水与 DB 记录一致性 | P0 |
| 并发支付 | 多人同时支付，无超卖/资金损失 | P0 |
| 支付金额越权 | 修改前端金额参数，验证后端以服务端金额为准 | P0 |

## L3 版本（商业化，时间待定）

### 功能：广告 / 品牌合作

| 测试项 | 说明 |
|---|---|
| 广告内容合规 | 广告文案无违禁词，符合广告法 |
| 用户数据不泄露给广告主 | 确认 SDK 数据上报范围 |
| 广告加载不影响核心链路性能 | 广告加载失败不阻塞 PK 流程 |

### 功能：健康数据对接（Apple Health / 微信运动）

| 测试项 | 说明 |
|---|---|
| 数据读取授权 | 用户授权后才能读取健康数据 |
| 数据准确性 | 步数、体重同步准确 |
| 断开授权 | 取消授权后停止读取 |
| 数据不上传至服务端 | 确认健康数据只在本地处理 |

### 功能：商业化积分/会员

| 测试项 | 说明 |
|---|---|
| 积分充值/消耗 | 资金安全，幂等性 |
| 会员权益差异 | 非会员/普通会员/高级会员功能边界 |
| 续费/到期处理 | 会员到期不清除历史数据 |

## 持续关注项（跨版本）

### 合规与安全

|---|---|---|
| PIPL 合规评审 | 每次数据字段变更后重新检查 | 每个版本 |
