## 一、部署方式

### 云函数部署（两种方式）

**方式一：微信开发者工具（推荐开发期使用）**
1. 右键云函数目录
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成，控制台查看日志

**方式二：CLI 部署（推荐 CI/CD 集成）**
```bash
npm install -g @cloudbase/cli
tcb login
tcb functions:deploy checkin-submit --env slim-pk-prod
```

### 定时触发器部署

`score-calc` 和 `pk-settle` 带定时触发器，部署时一并生效：

```json
// score-calc/config.json
{
  "triggers": [
    {
      "name": "dailyScoreCalc",
      "type": "timer",
      "config": "0 5 0 * * * *"
    }
  ]
}
```

验证：云开发控制台 → 云函数 → score-calc → 触发器，确认显示激活状态。

## 四、监控与日志

### 监控入口

| 监控项 | 查看位置 |
|--------|---------|
| 云函数调用量/成功率 | 云开发控制台 → 统计分析 |
| 云函数错误日志 | 云开发控制台 → 云函数 → 对应函数 → 日志 |
| 数据库读写量 | 云开发控制台 → 数据库 → 统计 |
| 云存储用量 | 云开发控制台 → 存储 → 概览 |
| 小程序性能 | 微信公众平台 → 小程序管理 → 数据分析 |
| 配额告警 | 微信云开发配置告警阈值，超 80% 发微信通知 |

### 日志规范

```typescript
// 统一日志格式
```

## 五、回滚方案

| 场景 | 回滚方式 | 耗时 |
|------|---------|------|
| 云函数 Bug | 云开发控制台 → 函数版本管理 → 切换历史版本 | 1 分钟 |
| 小程序代码 Bug | 微信开发者工具 → 历史版本 → 重新上传设为体验版 | 5 分钟 |
| 数据库误操作 | 云数据库控制台 → 回档（指定时间点） | 5-10 分钟 |
| 紧急止血 | 微信小程序后台 → 暂停体验版服务 | 即时 |

---

## 六、应急预案

| 故障 | 影响 | 应急动作 |
|------|------|---------|
| 云函数超时/报错 | 打卡/排行功能不可用 | 查日志 → 修复 → 重新部署 |
| 数据库不可写 | 打卡失败 | 等待平台自动恢复（< 1 分钟）|
| 云存储不可用 | 照片上传失败 | 降级：前端提示重试，允许暂时不上传（体验版容忍度高）|
| 免费配额超限 | API 调用失败 | 控制台查用量 → 清理无用数据或升级基础套餐 |
| 定时任务漏跑 | 积分未更新 | 手动触发 score-calc 云函数补跑（幂等，可重复执行）|

## 七、性能基线（L1）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 打卡接口响应时间 | < 2s | 含图片上传（前端直传，不走云函数）|
| 排行榜查询响应时间 | < 500ms | 读快照，O(n) 遍历 |
| 积分定时任务耗时 | < 30s | 5 人 × 21 天，串行计算 |
| 月 API 调用量 | < 5 万次 | 免费配额 20 万，安全边界 25% |
