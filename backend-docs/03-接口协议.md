# 接口协议（云函数 API）

> 版本：v1.0 | 更新：2026-07-14 | 维护：斯斯【后端】  
> 调用方式：`wx.cloud.callFunction({ name: '函数名', data: {...} })`  
> 鉴权：云函数自动注入 `OPENID`，前端无需传 token

---

## 通用约定

### 请求格式

```javascript
wx.cloud.callFunction({
  name: 'function-name',
  data: {
    // 业务参数直接放在 data 对象中
    roomId: 'xxx',
    type: 'weight'
  }
})
```

### 响应格式

**成功：**
```json
{
  "code": 0,
  "message": "ok",
  "data": { ... }
}
```

**失败：**
```json
{
  "code": 1001,
  "message": "参数校验失败：roomId 不能为空",
  "detail": "仅开发环境返回，生产不返回"
}
```

### 错误码速查

| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数校验失败 |
| 1002 | 未登录 |
| 1003 | 无权限操作 |
| 2001 | 房间不存在 |
| 2002 | 已在房间中 |
| 2003 | PK 已结束 |
| 2004 | PK 未开始 |
| 3001 | 今日已打卡（幂等场景不报错，仅体重覆盖） |
| 3002 | 打卡时间不在周期内 |
| 3003 | 体重超出合理范围 |
| 3004 | 体重变化异常 |
| 5000 | 系统内部错误 |

## 一、用户模块

### 1.1 auth-login — 微信登录

> 调用时机：每次进入小程序、token 过期时

**请求参数：**
```typescript
{
  code: string  // wx.login() 返回的 code，有效期 5 分钟
}
```

**响应：**
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "isNewUser": true,
    "user": {
      "userId": "_id 值",
      "openid": "不返回，内部使用",
      "nickname": "用户昵称",
      "avatarUrl": "头像 URL",
      "consentStatus": false
    }
  }
}
```

**业务逻辑：**
1. 用 `code` 换 `openid`（云函数内调用 `cloud.getWXContext()` 直接获得，无需调微信服务端接口）
2. 查 `users` 集合，`openid` 匹配则更新 `updated_at`，否则创建新记录
3. `isNewUser = true` 时前端跳转引导授权昵称/头像页

**注意：** 云函数场景下 `openid` 由微信系统注入，不依赖客户端传入，无法伪造。

---

### 1.2 user-update — 更新用户信息

**请求参数：**
```typescript
{
  nickname?: string       // 最大 20 字
  avatarUrl?: string      // 头像 URL（微信授权后获得）
  consentStatus?: boolean // 隐私同意（只能从 false → true，不可反向）
}
```

**响应：**
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "updated": true
  }
}
```

---

### 1.3 user-delete — 删除账号

> 合规要求，用户可主动删除所有数据

**请求参数：** 无（用 openid 标识当前用户）

**响应：**
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "deleted": true
  }
}
```

**业务逻辑：**
1. 删除 `users` 中该 openid 的记录
2. 删除 `participants` 中该 openid 的所有参与记录
3. 删除 `checkins` 中该 openid 的所有打卡记录
4. 删除 `score_snapshots` 中该 openid 的所有快照
5. 删除对应的云存储照片（异步，失败不影响主流程）
6. 以上操作尽量事务化（云数据库支持原子写，多集合操作顺序执行+记录日志）

## 二、房间模块

### 2.1 room-create — 创建 PK 房间

**请求参数：**
```typescript
{
  name: string        // 房间名称，1-20 字，必填
  startDate: string   // 'YYYY-MM-DD'，开始日期，必须 >= 今天
  endDate: string     // 'YYYY-MM-DD'，结束日期，须满足 7 <= 天数 <= 90
  initialWeight: number    // 发起人初始体重，单位 kg，精度 0.1，范围 30-300
  initialPhotoFileId: string  // 初始体重照片 fileID（前端上传后获得）
}
```

**响应：**
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "roomId": "xxxx",
    "inviteCode": "A3B2C1",
    "totalDays": 21
  }
}
```

**业务逻辑：**
1. 校验 `startDate` >= 今天（北京时间）
2. 校验 `endDate - startDate + 1` 在 [7, 90] 区间
3. 生成 6 位唯一邀请码（碰撞则重试最多 3 次）
4. 写 `pk_rooms`（status: `pending` 或 `active`，取决于 startDate 是否为今天）
5. 同步写 `participants`（发起人自动加入，存入初始体重）
6. 初始体重克数 = `Math.round(initialWeight * 1000)`

## 三、打卡模块

### 3.1 checkin-submit — 提交打卡

**请求参数：**
```typescript
// 体重打卡
{
  roomId: string
  type: 'weight'
  weight: number           // kg，精度 0.1，范围 30-300
  photoFileId: string      // 云存储 fileID，必填
}

// 饮食打卡
{
  roomId: string
  type: 'diet'
  done: boolean            // true=已控制饮食，false=未完成
}

// 运动打卡
{
  roomId: string
  type: 'exercise'
  done: boolean            // true=已运动，false=未完成
}
```

**响应：**
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "checkinId": "ck_xxxx",
    "todayStatus": {
      "weightDone": true,
      "dietDone": false,
      "exerciseDone": true
    }
  }
}
```

**防作弊校验（体重打卡）：**
1. 与初始体重偏差 > 40% → 拒绝，返回 `3004`
2. 与昨日体重差 > 5kg → 拒绝，返回 `3004`（首日打卡无此校验）
3. 范围校验：30kg ≤ weight ≤ 300kg，否则返回 `3003`

**幂等规则：**
- 体重打卡：当日已有记录则 **upsert 覆盖**（取最新值），返回 `code=0`
- 饮食/运动：当日已有记录则**直接返回现有状态**（幂等），返回 `code=0`

## 六、云存储规范

### 打卡照片路径

```
checkin-photos/{openid-hash8}/{roomId}/{YYYYMMDD}/{timestamp-ms}.jpg
```

其中 `openid-hash8` 为 openid 的 MD5 前 8 位（不直接暴露 openid）。

### 初始体重照片路径

```
initial-weights/{openid-hash8}/{roomId}.jpg
```

### 战报海报路径

```
posters/{roomId}/{openid-hash8}.png
```

### 访问规则

- 所有照片**私有存储**，不可公开访问
- 展示时通过 `wx.cloud.getTempFileURL` 获取有效期 2 小时的临时链接
- 不存储临时 URL（URL 会过期），只存 `fileID`
