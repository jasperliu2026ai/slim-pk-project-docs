## 二、云开发环境初始化

### 2.1 创建云开发环境

1. 打开微信开发者工具，用绑定小程序的微信号登录
2. 导入后端项目代码
3. 点击顶部工具栏「云开发」
4. 创建 3 个环境：

| 环境名 | EnvID（示例）| 用途 |
|--------|------------|------|
| dev | slim-pk-dev | 本地开发/联调 |
| staging | slim-pk-staging | CC 测试 |
| prod | slim-pk-prod | 生产环境 |

> 每个微信账号最多可创建 2 个免费环境；prod 可通过腾讯云控制台创建第 3 个

### 2.2 小程序端初始化云开发

在 `app.ts` 中：

```typescript
App({
  onLaunch() {
    wx.cloud.init({
      // 开发时切换 envId 即可切换环境
      env: 'slim-pk-dev',
      traceUser: true
    })
  }
})
```

## 三、云函数开发规范

### 3.1 本地开发工作流

```bash
# 进入某个云函数目录
cd cloudfunctions/checkin-submit

# 安装依赖
npm install

# TypeScript 编译（开发时）
npx tsc --watch

# 部署到云端（微信开发者工具右键上传即可）
# 或使用 CLI：
npx @cloudbase/cli functions:deploy checkin-submit --env slim-pk-dev
```

## 四、云数据库初始化

**第一次使用时，在云开发控制台手动创建集合和索引：**

### 4.1 创建集合

在云开发控制台 → 数据库 → 创建以下集合：

```
users
pk_rooms
participants
checkins
score_snapshots
settlements
```

### 4.2 创建索引

**users 集合：**
```
字段：openid，唯一索引：是
```

**pk_rooms 集合：**
```
字段：invite_code，唯一索引：是
字段：creator_openid，唯一索引：否
字段：status，唯一索引：否
```

**participants 集合：**
```
字段：room_id + openid，复合索引，唯一索引：是
字段：room_id，唯一索引：否
```

**checkins 集合（关键！）：**
```
字段：room_id + openid + date + type，复合索引，唯一索引：是
字段：room_id + date，复合索引，唯一索引：否
字段：openid + room_id，复合索引，唯一索引：否
```

**score_snapshots 集合：**
```
字段：room_id + openid + snapshot_date，复合索引，唯一索引：是
字段：room_id + snapshot_date，复合索引，唯一索引：否
```

**settlements 集合：**
```
字段：room_id + openid，复合索引，唯一索引：是
字段：room_id，唯一索引：否
```

## 五、本地联调

### 5.1 云函数本地调试

微信开发者工具支持云函数本地运行（不需要每次部署到云端）：

1. 右键云函数目录 → 「在本地调试该云函数」
2. 填入 event 参数（JSON 格式）
3. 点击运行，查看输出

### 5.2 与前端联调

1. 前端（睿睿）在 `app.ts` 中将 env 指向 `slim-pk-dev`
2. 后端函数部署到 `slim-pk-dev` 环境
3. 微信开发者工具中运行前端，直接调用云函数
4. 云函数日志在：云开发控制台 → 云函数 → 日志

### 5.3 测试数据初始化脚本

在云函数控制台可直接执行以下初始化脚本（开发环境用）：

```javascript

  data: {
```

## 六、代码规范与 Git 流程

### Git 分支约定

```
main              生产分支，只接受 PR 合并，不直接推
develop           开发主分支，日常联调
feature/xxx       功能开发分支（如 feature/checkin-submit）
fix/xxx           Bug 修复分支
```

### Commit Message 规范

```
feat: 新增打卡接口 checkin-submit
fix: 修复排行榜并列排名跳号问题
docs: 更新接口协议文档
chore: 升级 joi 版本至 17.9.0
refactor: 重构积分计算函数，抽取为工具函数
```

### 代码审查

- PR 合并到 develop/main 需要至少 1 人 Review
- 金额相关（押金/结算，L2 功能）的代码必须双人 Review

## 七、常见问题

**Q：云函数执行成功但前端拿不到数据？**  
A：检查 env 是否匹配，前端 `wx.cloud.init` 的 env 和后端部署的 env 必须一致。

**Q：云数据库写入报权限错误？**  
A：云函数调用数据库时使用管理员权限（`wx-server-sdk` 默认即管理员权限），不需要额外配置。前端直接调用数据库时才有权限限制。

**Q：TypeScript 编译后的 index.js 找不到？**  
A：确保 `tsc` 已执行，且 `outDir` 配置为 `.`（输出到当前目录）。部署时上传编译后的 `.js` 文件。

**Q：定时触发器没有生效？**  
A：确认 `config.json` 格式正确，且函数已重新部署（触发器配置随函数部署一起生效）。检查云开发控制台 → 云函数 → 对应函数 → 触发器。
