# 积分算法与结算逻辑

> 版本：v1.0 | 更新：2026-07-14 | 维护：斯斯【后端】

---

## 一、积分公式（当前版本）

```
综合积分（满分 100 分）=
  减重积分分项 × 70%
  + 打卡积分分项 × 15%
  + 饮食积分分项 × 8%
  + 运动积分分项 × 7%
```

### 各维度计算

```
减重积分分项（0-100）= min( (初始体重 - 最新体重) / 初始体重 × 100 × K, 100 )

打卡积分分项（0-100）= 体重打卡天数 / PK总天数 × 100

饮食积分分项（0-100）= 饮食打卡完成天数 / 已过天数 × 100

运动积分分项（0-100）= 运动打卡完成天数 / 已过天数 × 100
```

**参数说明：**
- **K 值**：减重放大系数，PRD 暂定 K=6（待 v_juvnliu 最终确认）
  - K=6 含义：减重 10% → 减重积分 60 分；减重约 16.7% → 满分 100 分
  - K=1 含义：减重 10% → 减重积分 10 分（激励效果弱）
- **已过天数**：从 PK 开始日到当前日期（含）的天数
- **PK 总天数**：`endDate - startDate + 1`
- 打卡积分分项分母用**总天数**（激励全程不落），饮食/运动分项分母用**已过天数**（避免前期膨胀）

---

## 二、存储与精度规范

| 环节 | 精度规则 |
|------|---------|
| 数据库存储 | `total_score × 100`，取整存储（整数型，无小数精度损耗） |
| 计算过程 | 用 JavaScript `Number` 进行浮点计算，最终 `Math.round(score * 100)` |
| 前端展示 | `db_value / 100`，`toFixed(2)` 保留 2 位，四舍五入 |
| 体重存储 | 单位克（整数），`Math.round(kg * 1000)` |

## 三、计算示例

**场景**：PK 21 天，已过 10 天，K=6

| 数据 | 值 |
|------|----|
| 初始体重 | 80kg（80000 克）|
| 当前最新体重 | 77kg（77000 克）|
| 体重打卡天数 | 8 天 |
| 饮食打卡完成天数 | 7 天 |
| 运动打卡完成天数 | 5 天 |

```javascript
// 减重积分分项
const weightLossPct = (80000 - 77000) / 80000 * 100   // = 3.75%
const weightLossItem = Math.min(weightLossPct * 6, 100) // = 22.5

// 打卡积分分项（分母 = PK总天数）
const checkinItem = 8 / 21 * 100   // = 38.10

// 饮食积分分项（分母 = 已过天数）
const dietItem = 7 / 10 * 100      // = 70.0

// 运动积分分项（分母 = 已过天数）
const exerciseItem = 5 / 10 * 100  // = 50.0

// 综合积分
const totalScore =
  weightLossItem * 0.70 +  // 22.5 × 0.70 = 15.75
  checkinItem * 0.15 +      // 38.10 × 0.15 = 5.715
  dietItem * 0.08 +          // 70.0 × 0.08 = 5.6
  exerciseItem * 0.07        // 50.0 × 0.07 = 3.5
// = 30.565

// 数据库存储
const dbValue = Math.round(totalScore * 100)  // = 3057
// 展示值
const displayValue = (dbValue / 100).toFixed(2)  // = "30.57"
```

## 四、T+1 定时任务（score-calc）

### 触发配置

```json
{
  "triggers": [
    {
      "name": "dailyScoreCalc",
      "type": "timer",
      "config": "0 5 0 * * * *"
    }
  ]
}
```

Cron 七位格式：`秒 分 时 日 月 星期 年`  
`0 5 0 * * * *` = 每天 **00:05:00**（北京时间，云开发定时器默认 UTC+8）

### 执行逻辑

```typescript
// score-calc/index.ts 伪代码
export const main = async () => {
  const today = getBeijingDateStr()         // 'YYYY-MM-DD'
  const yesterday = getYesterdayStr()       // T-1

  // 1. 查所有进行中的房间
  const activeRooms = await db.collection('pk_rooms')
    .where({ status: 'active' })
    .get()

  for (const room of activeRooms.data) {
    // 2. 查该房间所有活跃参与者
    const members = await db.collection('participants')
      .where({ room_id: room._id, is_active: true })
      .get()

    for (const member of members.data) {
      // 3. 计算积分（基于截至昨日的全量打卡记录）
      const score = await calcScore(member, room, yesterday)

      // 4. upsert 写入快照（幂等，相同 room_id+openid+date 则覆盖）
      await upsertSnapshot(room._id, member.openid, today, score)
    }

    // 5. 检查房间是否到期
    if (yesterday === room.end_date) {
      await settleRoom(room)
    }
  }

  console.log(`[score-calc] 完成，处理房间数：${activeRooms.data.length}`)
}
```

### 幂等保障

- `score_snapshots` 上有唯一复合索引 `(room_id, openid, snapshot_date)`
- 写入时用 `upsert`（存在则更新，不存在则插入）
- 即使定时任务重跑，结果一致，不会产生重复记录

## 五、排行榜并列处理

```typescript
// 并列排名：分数相同则相同名次，下一位不跳号
// 示例：[100, 100, 80] → rank=[1, 1, 2]（不是 1, 1, 3）

function calcRanks(list: { totalScore: number }[]) {
  const sorted = [...list].sort((a, b) => b.totalScore - a.totalScore)
  let rank = 1
  return sorted.map((item, idx) => {
    if (idx > 0 && sorted[idx - 1].totalScore !== item.totalScore) {
      // 与上一名分数不同，更新 rank 为当前实际位置
      rank = idx + 1
    }
    return { ...item, rank }
  })
}
```

## 六、PK 结算（pk-settle）

### 触发时机

1. **自动触发**：`score-calc` 检测到房间 `end_date = yesterday` 时，自动调用结算
2. **前端拉取**：PK 状态为 `ended` 时，前端请求 `pk-settle` 获取战报数据

### 结算步骤

```
1. 读取最终快照
   → 查 score_snapshots，roomId + snapshot_date = end_date
   → 若无快照（用户从未打卡）则 totalScore = 0

2. 计算最终排名
   → 用并列排名算法（同上）
   → 已退出成员（isActive=false）：积分冻结在退出时，排在末位

3. 写 settlements 集合
   → (room_id, openid) 唯一，幂等写入

4. 更新房间状态
   → pk_rooms.status = 'ended'

5. 返回战报数据给前端
```

### 战报数据格式

```typescript
interface SettlementResult {
  rank: number
  nickname: string
  avatarUrl: string
  finalScore: number         // 展示值，2 位小数
  weightLossKg: number       // 减重 kg（负数 = 增重）
  weightLossPercent: number  // 减重百分比（2 位小数）
  totalCheckinDays: number
  totalDays: number
}
```

## 七、边界 Case 处理

| Case | 处理方式 |
|------|---------|
| 用户从未打卡体重 | 减重积分 = 0，打卡积分 = 0，饮食/运动积分正常计算 |
| 用户中途退出 | 退出时冻结积分快照，此后不再更新；排行榜显示"已退出"，置末位 |
| PK 第一天（elapsed_days=1） | 饮食/运动积分正常计算，打卡积分 = 0 或 1/totalDays |
| 减重超过 K 值上限 | `min(score, 100)` 封顶，最高 100 分 |
| 增重（体重上升） | 减重积分 = 0（不扣分，仅无加分） |
| 并列最后一名 | 相同积分同名次，战报上显示共同排名 |
| 定时任务迟到（异常重跑）| 幂等写入，覆盖已有快照，结果一致 |
