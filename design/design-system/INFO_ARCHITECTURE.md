> 版本：v0.1（初稿）  
> 更新：2026-07-14

## 一、Tab 结构（一级导航）

小程序底部导航只保留 **4 个 Tab**，克制不堆：

```
┌────────┬────────┬────────┬────────┐
│  今日   │   PK   │  数据   │   我   │
│ Today  │ Battle │ Stats  │  Me   │
└────────┴────────┴────────┴────────┘
```

| Tab | 定位 | 核心动作 |
|---|---|---|
| **今日** | 每日打开的主页，聚焦"今天要做什么" | 打卡（体重/饮食/运动）、看当前 PK 状态、每日鼓励 |
| **PK** | 社交竞争主战场 | 创建/加入 PK、PK 详情、排行榜、历史战绩 |
| **数据** | 长期成长的可视化 | 体重曲线、周报、月报、勋章墙 |
| **我** | 个人中心 | 资料、目标设置、隐私、消息、设置 |

## 二、核心用户旅程（User Journeys）

### 🌅 旅程 A：早晨打卡（高频，日均 1 次）
```
打开小程序 → [今日] 见到问候语 + 今日打卡卡片
  → 点击「体重打卡」 → 数字输入（大键盘、数字动画）
  → 提交 → 【打卡成功页】高光时刻 ✨
  → 回到 [今日]，卡片状态变为"已打卡" + 显示今日趋势
```
**关键页面**：今日主页 / 打卡输入页 / **打卡成功页（高光页）**

### ⚔️ 旅程 B：参与 PK（核心留存点）
```
[PK] Tab → 「进行中」列表 → 点击某场 PK
  → PK 详情页：奖池 / 剩余天数 / 参赛者进度条排行
  → 底部固定「今日打卡」按钮
  → 打卡后返回，看到自己名次跳动上升
```
**关键页面**：PK 列表页 / PK 详情页 / **PK 结算页（高光页）**

### 📊 旅程 C：查看进度（周/月度）
```
[数据] Tab → 顶部大数字（当前体重 vs 目标） + 曲线图
  → 下滑：本周概览卡 / 本月概览卡 / 勋章墙
  → 点击「查看周报」→ 【周报页】(可分享的高光页)
```
**关键页面**：数据主页 / **周报页（高光页）** / 勋章详情

### 🆘 旅程 D：异常态（不可忽视）
- 目标不合理（BMI 过低 / 一周想减 10 斤） → 温和劝阻页
- 连续 3 天未打卡 → 陪伴式召回，不用"你失败了"式语言
- PK 掉队 → 中性表达 + 队友加油入口，不做羞辱

## 三、页面清单（P0 阻塞上线）

| 页面 | 优先级 | 是否高光页 | 备注 |
|---|---|---|---|
| 引导 / 目标设定（首次） | P0 | – | 3 步走完，问动机而非只问体重 |
| 今日主页 | P0 | ⭐ | 打开即用，情感锚点 |
| 体重打卡 | P0 | – | 数字键盘 + 单位切换 |
| **打卡成功页** | P0 | 🔥 | 值得截图，触发分享 |
| PK 列表 | P0 | – | 「进行中 / 报名中 / 已结束」三分类 |
| PK 详情 | P0 | ⭐ | 实时进度、队友互动 |
| **PK 结算页** | P0 | 🔥 | 值得截图，胜负都要有面 |
| 数据主页（曲线） | P0 | – | 体重曲线是核心 |
| **周报页** | P0 | 🔥 | 值得截图，产品自传播 |
| 个人中心 | P0 | – | 目标、隐私、设置入口 |

## 六、页面之间的高光钩子（关键）

**产品自传播依赖三张页**：
1. **打卡成功页**：每次成功打卡都是"值得截图"的瞬间
2. **PK 结算页**：胜负都做得体面（胜利有仪式感、失败有陪伴感）
3. **周报页**：每周一次的仪式，鼓励用户主动分享

这三张页在视觉预算上**允许更多动效和高光**（大数字、光晕、粒子克制版），其他页保持克制。
