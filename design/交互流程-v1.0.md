# 交互流程 v1.0

## 1. 创建 PK 流程

首页点击创建 PK → 填写基础信息 → 设置周期、人数、规则 → 规则与隐私确认 → 创建成功 → 分享邀请。

关键校验：PK 名称不能为空，建议 4 到 16 字；周期需在产品规则范围内；若涉及押金或奖金池，必须展示规则入口、退款规则、争议处理入口。

## 2. 加入 PK 流程

打开邀请卡 → 加入页 → 判断是否登录 → 查看规则与隐私 → 判断房间是否可加入 → 确认加入 → 进入 PK 详情。

异常态：房间已满、已结束、已解散、未成年人或特殊人群准入提示。

## 3. 每日打卡流程

首页或 PK 详情点击今日打卡 → 输入体重 → 可选上传照片 → 选择照片可见范围 → 提交 → 成功页 → 返回 PK 详情或分享。

体验细节：二次进入打卡页时恢复草稿；提交中禁用按钮并展示进度，不清空输入；今日已打卡再次进入时展示记录。

## 4. 排行榜流程

进入 PK 详情 → 排行榜模块曝光 → 展示 Top3 和我的位置 → 点击查看全部 → 排行榜全页 → 今日或累计筛选。

隐私策略：默认只展示积分、相对进度、连续打卡。不展示绝对体重、体脂率、身体照片缩略图。

## 5. 结算与分享流程

PK 到期 → 生成结算 → 结算页 → 生成分享海报 → 选择展示数据 → 保存或分享。

分享卡默认字段：PK 名称、完赛或坚持天数、排名或荣誉称号、相对成长信息、小程序码。精确体重变化和详细曲线默认关闭。

## 6. 退出和解散流程

参与者退出：说明退出影响 → 二次确认 → 退出成功 → 保留历史只读。
发起人解散：说明成员影响 → 输入确认 → 解散成功 → 通知成员。
不做诱导挽留，不隐藏退出入口。
