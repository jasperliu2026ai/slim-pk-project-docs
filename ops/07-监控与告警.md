# 07 · 监控与告警

> 版本：v1.0 | 2026-07-14

## 1. SLO 定义

| 指标 | SLO 目标 | 告警阈值 | 告警级别 |
|------|----------|----------|----------|
| API 可用率 | ≥ 99.9% / 月 | < 99.5% 持续5分钟 | P1 |
| 打卡成功率 | ≥ 99.5% | < 98% | P1 |
| 支付成功率 | ≥ 99% | < 98% | P0 |
| API P99 延迟 | ≤ 500ms | > 1000ms 持续5分钟 | P2 |
| COS 上传成功率 | ≥ 99% | < 95% | P1 |

## 2. 基础设施监控指标

| 指标 | 告警阈值 | 处置 |
|------|----------|------|
| CVM CPU 使用率 | > 80% 持续10分钟 | 排查是否需要扩容 |
| CVM 内存使用率 | > 85% | 检查内存泄漏 |
| CVM 磁盘使用率 | > 80% | 清理日志/扩容磁盘 |
| MySQL 连接数 | > 150（上限200） | 检查连接池设置 |
| MySQL 慢查询数 | > 10条/分钟 | 分析慢查询日志 |
| Redis 内存使用率 | > 80% | 检查 Key 过期策略 |
| Redis 命中率 | < 80% | 分析缓存策略 |

## 4. 日志体系

### 4.1 日志分级

| 级别 | 使用场景 | 保留时长 |
|------|----------|----------|
| ERROR | 异常、崩溃、业务失败 | 90 天 |
| WARN | 慢请求、重试、降级 | 30 天 |
| INFO | 关键业务操作（打卡/支付/登录） | 30 天 |
| DEBUG | 调试信息 | 7 天（仅 dev/staging） |

### 4.2 日志格式（JSON 结构化）

```json
{
  "timestamp": "2026-07-14T16:00:00.000+08:00",
  "level": "INFO",
  "service": "slim-pk-api",
  "env": "prod",
  "traceId": "abc-123-def",
  "userId": "hash:abc123",
  "action": "checkin.upload",
  "duration": 245,
  "status": "success",
  "msg": "打卡上传成功"
}
```

> ⚠️ **日志脱敏规则**：体重数值、手机号、支付金额、openid **不得出现在日志明文**。userId 只记录哈希。

### 4.3 CLS 日志投递（腾讯云）

```
LogListener 安装于每台 CVM
收集路径：/opt/slim-pk/app/shared/logs/*.log
投递到：日志集 slim-pk-prod
索引字段：level, action, userId, traceId, status, duration
```

## 5. 告警分级与通道

| 级别 | 定义 | 响应时间 | 通知方式 |
|------|------|----------|----------|
| P0 | 支付异常、全站不可用 | 立即 | **电话 + 短信 + 企微** |
| P1 | 核心功能降级（打卡失败率高） | 5 分钟内 | **短信 + 企微** |
| P2 | 性能劣化、非核心异常 | 30 分钟内 | 企微 |
| P3 | 轻微异常、资源预警 | 工作时间处理 | 企微 |

## 6. 关键告警规则（示意）

```yaml
- name: "API 错误率突增"
  condition: "5xx_rate > 1% for 2min"
  level: P1
  channels: [sms, wecom]

- name: "支付回调异常"
  condition: "payment_callback_fail > 0 for 1min"
  level: P0
  channels: [phone, sms, wecom]

- name: "MySQL 连接数告警"
  condition: "db_connections > 150 for 5min"
  level: P2
  channels: [wecom]

- name: "磁盘使用率告警"
  condition: "disk_usage > 80%"
  level: P3
  channels: [wecom]
```

## 7. 值班响应规范

| 时间段 | 响应要求 |
|--------|----------|
| 工作日 09:00–22:00 | P0 5分钟响应，P1 15分钟响应 |
| 夜间 22:00–09:00 | P0 15分钟响应（电话告警），P1 次日处理 |
| 节假日 | 同夜间，P0 保持电话告警 |
