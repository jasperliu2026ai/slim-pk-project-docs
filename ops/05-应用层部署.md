# 05 · 应用层部署

> 版本：v1.0 | 2026-07-14

## 1. CVM 初始化（每台执行）

### 1.1 基础软件安装

```bash
# 更新系统（Ubuntu 22.04）
apt update && apt upgrade -y

# 安装 Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 安装 PM2（进程管理）
npm install -g pm2

# 安装 Nginx
apt install -y nginx

# 安装常用工具
apt install -y git htop curl wget unzip jq
```

### 1.2 目录结构

```
/opt/slim-pk/
├── app/
│   ├── current/      ← 当前运行版本（软链接）
│   ├── releases/     ← 历史版本（保留最近5个）
│   └── shared/
│       ├── .env      ← 从 SSM 生成，启动时注入
│       └── logs/     ← 应用日志
└── scripts/
    ├── deploy.sh     ← 部署脚本
    ├── rollback.sh   ← 回滚脚本
    └── gen-env.sh    ← 从 SSM 生成 .env
```

### 1.3 PM2 配置（ecosystem.config.js）

```javascript
module.exports = {
  apps: [{
    name: 'slim-pk-api',
    script: '/opt/slim-pk/app/current/src/app.js',
    instances: 2,           // 每台 CVM 2个进程
    exec_mode: 'cluster',
    env_production: {
      NODE_ENV: 'production',
      PORT: 3000,
    },
    error_file: '/opt/slim-pk/app/shared/logs/err.log',
    out_file: '/opt/slim-pk/app/shared/logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    max_memory_restart: '1G',
  }]
};
```

## 2. Nginx 配置

```nginx
# /etc/nginx/sites-available/slim-pk.conf
upstream slim_pk_backend {
    server 127.0.0.1:3000;
    keepalive 32;
}

server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;

    client_max_body_size 10m;   # 打卡照片上限
    proxy_read_timeout 30s;
    proxy_connect_timeout 5s;

    location /api/ {
        proxy_pass http://slim_pk_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /health {
        proxy_pass http://slim_pk_backend;
        access_log off;
    }
}
```

## 3. 部署脚本（deploy.sh）

```bash
#!/bin/bash
set -e
VERSION=$1
RELEASE_DIR="/opt/slim-pk/app/releases/$VERSION"

echo "[1/6] 拉取代码 $VERSION ..."
git clone --depth 1 --branch "$VERSION" \
  https://github.com/your-org/slim-pk-backend.git "$RELEASE_DIR"

echo "[2/6] 安装依赖 ..."
cd "$RELEASE_DIR" && npm ci --production

echo "[3/6] 生成配置 ..."
/opt/slim-pk/scripts/gen-env.sh > /opt/slim-pk/app/shared/.env

echo "[4/6] 切换软链接 ..."
ln -sfn "$RELEASE_DIR" /opt/slim-pk/app/current

echo "[5/6] 优雅重启 ..."
pm2 reload slim-pk-api

echo "[6/6] 健康检查 ..."
sleep 5
curl -sf http://localhost:3000/health || { pm2 reload slim-pk-api; echo "健康检查失败，已触发回滚"; exit 1; }

echo "部署完成: $VERSION"
```

## 4. 回滚脚本（rollback.sh）

```bash
#!/bin/bash
set -e
PREV=$(ls -t /opt/slim-pk/app/releases/ | sed -n '2p')
if [ -z "$PREV" ]; then
  echo "没有可回滚的历史版本"
  exit 1
fi
echo "回滚到: $PREV"
ln -sfn "/opt/slim-pk/app/releases/$PREV" /opt/slim-pk/app/current
pm2 reload slim-pk-api
sleep 3
curl -sf http://localhost:3000/health && echo "回滚成功" || echo "回滚后健康检查失败，请人工介入"
```

## 5. CLB 健康检查配置

| 参数 | 值 |
|------|----|
| 检查路径 | GET /health |
| 正常状态码 | 200 |
| 检查间隔 | 5 秒 |
| 超时时间 | 2 秒 |
| 健康阈值 | 连续 2 次正常才标记健康 |
| 不健康阈值 | 连续 3 次失败则摘除节点 |

## 6. 开机自启配置

```bash
# 将 PM2 进程持久化到系统服务
pm2 startup systemd

```
