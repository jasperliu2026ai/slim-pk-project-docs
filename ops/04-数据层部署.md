# 04 · 数据层部署

> 版本：v1.0 | 2026-07-14

## 1. MySQL（TencentDB for MySQL）

### 1.1 规格配置

| 环境 | 规格 | 存储 | 版本 | 高可用 |
|------|------|------|------|--------|
| dev/staging | 1核2G | 50G SSD | 8.0 | 基础单机版 |
| prod | 2核4G | 200G SSD | 8.0 | **高可用版（主从）** |

### 1.2 初始化步骤

```sql
-- 1. 创建业务数据库
CREATE DATABASE slim_pk CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 2. 创建应用账号（最小权限）
CREATE USER 'slimpk_app'@'%' IDENTIFIED BY '<从SSM读取>';
GRANT SELECT, INSERT, UPDATE, DELETE ON slim_pk.* TO 'slimpk_app'@'%';

-- 3. 创建只读账号（供测试/数据分析）
CREATE USER 'slimpk_readonly'@'%' IDENTIFIED BY '<从SSM读取>';
GRANT SELECT ON slim_pk.* TO 'slimpk_readonly'@'%';

-- 4. 禁止 root 远程登录
UPDATE mysql.user SET Host='localhost' WHERE User='root';
FLUSH PRIVILEGES;
```

### 1.3 参数调优

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| innodb_buffer_pool_size | 内存的 60% | 热数据缓存 |
| max_connections | 200 | 小程序并发场景 |
| slow_query_log | ON | 开启慢查询记录 |
| long_query_time | 1 | 超过1秒记录 |
| binlog_expire_logs_seconds | 604800 | Binlog 保留7天 |

### 1.4 备份策略

- **自动备份**：每天 02:00–04:00（低峰期），保留 7 天
- **手动快照**：每次大版本发布前，手动触发一次备份
- **Binlog**：开启，保留 7 天，支持按时间点恢复（PITR）

### 1.5 核心数据表清单

| 表名 | 说明 | 关键索引 |
|------|------|----------|
| users | 用户信息（openid、昵称） | UNIQUE(openid) |
| pk_rooms | PK 房间（周期、押金、状态） | INDEX(status, end_date) |
| room_members | 房间成员（初始体重，加密存储） | INDEX(room_id, user_id) |
| daily_checkins | 每日打卡记录 | INDEX(room_id, user_id, checkin_date) |
| weight_records | 体重记录（字段加密） | INDEX(user_id, record_date) |
| rank_snapshots | 排行榜快照（T+1更新） | INDEX(room_id, snapshot_date) |
| payments | 支付/退款流水 | UNIQUE(out_trade_no) |

## 2. Redis（腾讯云 Redis）

### 2.1 规格配置

| 环境 | 规格 | 版本 | 模式 |
|------|------|------|------|
| dev | 256M | 5.0 | 单机 |
| prod | **4G** | 5.0 | **主从版** |

### 2.2 Key 设计规范

```
slim-pk:{env}:user:{uid}:session          → 用户 Session（TTL: 7天）
slim-pk:{env}:room:{rid}:rank             → 排行榜 ZSet（按积分排序）
slim-pk:{env}:room:{rid}:checkin:{date}   → 当日打卡用户集合（TTL: 2天）
slim-pk:{env}:user:{uid}:checkin:{date}   → 打卡状态（TTL: 2天）
slim-pk:{env}:lock:payment:{tradeNo}      → 支付幂等锁（TTL: 30min）
```

### 2.3 内存使用预估

| Key 类型 | 预估数量 | 单条大小 | 总计 |
|----------|----------|----------|------|
| 用户 Session | 5000 DAU | ~200B | ~1MB |
| 排行榜 ZSet | 500 房间 | ~2KB/房间 | ~1MB |
| 打卡状态 | 5000 条 | ~50B | ~0.25MB |
| **合计** | | | **~5MB（远低于4G上限）** |

### 2.4 运维配置

```
maxmemory-policy: allkeys-lru    # 内存满时 LRU 淘汰
appendonly: yes                   # 开启 AOF 持久化
save 900 1                        # 900秒内1次写操作则 RDB 快照
```

## 3. COS 对象存储初始化

### 3.1 桶规划

```bash
# 生产环境三个桶
slim-pk-prod-uploads-{appid}    # 用户上传（私有）
slim-pk-prod-assets-{appid}     # 静态资源（公有读，绑CDN）
slim-pk-prod-logs-{appid}       # 日志归档（私有）
```

### 3.2 生命周期策略（用户上传桶）

```json
{
  "Rules": [
    {
      "ID": "move-to-low-freq",
      "Status": "Enabled",
      "Filter": {"Prefix": "checkin/"},
      "Transition": {"Days": 90, "StorageClass": "STANDARD_IA"}
    },
    {
      "ID": "move-to-archive",
      "Status": "Enabled",
      "Filter": {"Prefix": "checkin/"},
      "Transition": {"Days": 365, "StorageClass": "ARCHIVE"}
    }
  ]
}
```

### 3.3 目录结构规范

```
slim-pk-prod-uploads/
├── checkin/{year}/{month}/{day}/{uid}_{ts}.jpg   ← 打卡照片
├── weight/{uid}/{ts}.jpg                          ← 体重截图
└── avatar/{uid}.jpg                               ← 用户头像

slim-pk-prod-assets/
├── static/          ← 小程序静态资源
├── activity/        ← 活动素材
└── default-avatar.jpg
```
