# 10 · 运维操作手册（日常 SOP）

> 版本：v1.0 | 2026-07-14
> 每次操作须留变更记录，见末尾模板。

## 1. 服务状态查看

```bash
# PM2 进程状态
pm2 list
pm2 status slim-pk-api

# 实时日志
pm2 logs slim-pk-api --lines 100

# 错误日志
pm2 logs slim-pk-api --err --lines 200

# 系统资源
htop
df -h       # 磁盘
free -m     # 内存
```

## 2. 服务重启

```bash
# 优雅重启（推荐，不中断请求）
pm2 reload slim-pk-api

# 强制重启（慎用，会中断请求）
pm2 restart slim-pk-api

# 重启后验证
curl -sf http://localhost:3000/health && echo "OK"
```

## 3. Nginx 操作

```bash
nginx -t               # 检查配置语法
nginx -s reload        # 优雅重载配置

tail -100 /var/log/nginx/access.log    # 访问日志
tail -100 /var/log/nginx/error.log     # 错误日志

# 按 IP 统计请求量（找异常流量）
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20
```

## 4. 数据库运维

### 4.1 连接数据库（通过堡垒机 SSH 隧道）

```bash
# 建立隧道（本地 3307 → 远程 3306）
ssh -L 3307:mysql-host:3306 bastion-host -N &

# 连接
mysql -h 127.0.0.1 -P 3307 -u slimpk_app -p slim_pk
```

### 4.2 常用 SQL 巡检

```sql
-- 当前连接数
SHOW STATUS LIKE 'Threads_connected';

-- 活跃连接（排查慢查询/锁）
SHOW PROCESSLIST;

-- 今日打卡量
SELECT DATE(created_at) AS date, COUNT(*) AS cnt
FROM daily_checkins
WHERE created_at >= CURDATE()
GROUP BY DATE(created_at);

-- 各表大小
SELECT table_name,
  ROUND((data_length + index_length) / 1024 / 1024, 2) AS size_mb
FROM information_schema.tables
WHERE table_schema = 'slim_pk'
ORDER BY size_mb DESC;
```

### 4.3 手动触发备份（发版前执行）

```bash
# 腾讯云控制台 → TencentDB → 实例详情 → 备份恢复 → 立即备份
# 或通过 tccli
tccli cdb CreateBackup --InstanceId cdb-xxxxxxxx --BackupMethod logical
```

## 5. Redis 运维

```bash
# 连接
redis-cli -h redis-host -p 6379 -a <密码>

# 查看内存
INFO memory

# 查看连接数
INFO clients

# 扫描某类 key（生产用 SCAN，不用 KEYS）
SCAN 0 MATCH "slim-pk:prod:room:*" COUNT 100

# 查看排行榜
ZREVRANGE "slim-pk:prod:room:123:rank" 0 9 WITHSCORES

# 清除指定房间排行榜缓存
DEL "slim-pk:prod:room:123:rank"

# ⚠️ 禁止在生产执行 FLUSHDB / FLUSHALL
```

## 7. CLS 日志检索（腾讯云）

```
# 常用检索语句（CLS 控制台 → 检索分析）

查找 ERROR 日志：
  level: ERROR | limit 100

查找支付失败：
  action: "payment.callback" AND status: "fail"

查找慢请求（超过1秒）：
  duration > 1000

统计接口错误率：
  * | SELECT action, count(*) total,
      sum(if(status='fail',1,0)) fail_cnt,
      sum(if(status='fail',1,0))/count(*)*100 fail_rate
    GROUP BY action ORDER BY fail_rate DESC
```

## 8. 扩容操作 SOP

### CVM 垂直扩容（需重启，选低峰期）

```
Step 1  确认需要扩容（CPU/内存持续 > 80%）
Step 2  通知相关同学，选凌晨 2-5 点窗口
Step 3  先对 CVM-02 操作（CLB 只走 CVM-01）
Step 4  CVM-02 升配完成、开机、验证健康
Step 5  再对 CVM-01 操作（CLB 只走 CVM-02）
Step 6  CVM-01 升配完成、开机、两台均验证正常
```

### Redis 扩容（在线，不中断服务）

```
Step 1  控制台 → Redis → 变更配置
Step 2  扩容期间服务不中断
Step 3  扩容后验证命中率和连接数正常
```

```

```
