# 03 · 网络与安全基线

> 版本：v1.0 | 2026-07-14

## 1. VPC 网络规划

```
VPC: slim-pk-vpc（172.16.0.0/16）广州区
│
├── 业务子网 A（172.16.1.0/24）可用区A  ← CVM-01、CLB
├── 业务子网 B（172.16.2.0/24）可用区B  ← CVM-02（跨AZ冗余）
├── 数据子网 A（172.16.3.0/24）可用区A  ← MySQL 主节点
└── 数据子网 B（172.16.4.0/24）可用区B  ← MySQL 从节点、Redis
```

> 核心原则：**数据子网不分配公网 IP，禁止出方向访问公网**。

## 2. 安全组规则

### 2.1 CLB 安全组（对外开放）

| 方向 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS 业务流量 |
| 入站 | TCP | 80 | 0.0.0.0/0 | HTTP（重定向至 443） |
| 出站 | ALL | ALL | VPC 内部 | 转发到 CVM |

### 2.2 CVM 安全组

| 方向 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站 | TCP | 3000 | CLB 安全组 | 接收 CLB 转发 |
| 入站 | TCP | 22 | 堡垒机 IP | SSH（仅堡垒机） |
| 出站 | TCP | 3306 | 数据子网 | 连接 MySQL |
| 出站 | TCP | 6379 | 数据子网 | 连接 Redis |
| 出站 | TCP | 443 | 0.0.0.0/0 | 调用微信 API/COS |

### 2.3 数据库安全组

| 方向 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站 | TCP | 3306 | CVM 安全组 | 仅应用层可访问 |
| 入站 | TCP | 6379 | CVM 安全组 | 仅应用层可访问 |
| 出站 | DENY | ALL | 0.0.0.0/0 | 禁止数据库出流量 |

## 3. 堡垒机配置

- **方案**：腾讯云 TAT（无代理运维）或独立跳板机 CVM（小规模用跳板机即可）
- **规则**：生产 CVM 的 22 端口只对堡垒机开放，**任何人 SSH 生产机必须走堡垒机**
- **审计**：所有 SSH 操作记录到 CLS 日志，保留 90 天

## 4. 密钥与配置管理

| 信息类型 | 存放位置 | 禁止位置 |
|----------|----------|----------|
| 腾讯云 SecretId/Key | 腾讯云 SSM 参数仓库 | 代码库、硬编码 |
| 数据库密码 | SSM 参数仓库 + 启动脚本读取 | .env 文件入库 |
| 微信 AppSecret | SSM 参数仓库 | 代码库 |
| JWT Secret | SSM 参数仓库 | 代码库 |
| COS 临时密钥 | 后端运行时调用 STS 生成 | 下发到前端持久化 |

**应用启动流程**：
```bash
# 启动时从 SSM 读取密钥，注入环境变量，不落磁盘
node app.js --ssm-prefix=/slim-pk/prod/
```

## 5. HTTPS 与证书

- **证书方案**：Let's Encrypt 免费 DV 证书，acme.sh 自动续期（90天，cron 提前30天触发）
- **备用方案**：腾讯云 DV 免费证书（1年，到期前邮件提醒）
- **强制 HTTPS**：Nginx 80 端口统一 301 跳转到 443

## 6. COS 桶权限策略

| 桶名 | 权限 | 访问方式 |
|------|------|----------|
| slim-pk-prod-assets（静态资源） | 公有读 | CDN 加速分发 |
| slim-pk-prod-uploads（用户上传） | 私有 | 后端持有密钥上传；读取用临时签名 URL（15分钟有效） |
| slim-pk-prod-logs（日志归档） | 私有 | 仅后端写入，运维只读 |

**CDN 防盗链配置**：
- Referer 白名单：`*.yourdomain.com`、`servicewechat.com`（微信域名）
- 空 Referer：禁止
