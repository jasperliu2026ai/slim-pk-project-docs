# 08 — 埋点规范

> 版本：v1.0 | 日期：2026-07-14 | 作者：睿睿【前端】

---

## 一、埋点原则

1. **数据最小化**：只收集业务必需的数据，不上报体重原始值等高度敏感字段
2. **脱敏规则**：手机号、身份证不上报；体重字段上报「区间档位」而非精确值
3. **不影响主流程**：埋点失败不影响用户操作，catch 静默处理
4. **统一入口**：所有埋点通过 `utils/track.ts` 发出，不散落各处直接调用 `wx.reportEvent`

## 二、公共属性（每个事件自动携带）

```typescript
interface TrackCommonProps {
  app_version: string;         // 小程序版本号（来自 __APP_VERSION__）
  platform: string;            // iOS / Android
  model: string;               // 设备型号
  system: string;              // 系统版本
  network_type: string;        // wifi / 4g / 3g / 2g / unknown
  session_id: string;          // 本次启动的 UUID（onLaunch 时生成）
  user_id: string;             // 用户 ID（已登录时有）
  openid: string;              // openid（已登录时有）
}
```

## 三、utils/track.ts 封装

```typescript
// utils/track.ts

let sessionId = '';

export function initTrack() {
  sessionId = generateUUID();
}

function getCommonProps(): Record<string, any> {
  const sysInfo = wx.getSystemInfoSync();
  const user = getApp().globalData.userInfo;
  return {
    app_version: '__APP_VERSION__', // CI 注入
    platform: sysInfo.platform,
    model: sysInfo.model,
    system: sysInfo.system,
    network_type: getNetworkType(), // 缓存值，避免每次异步获取
    session_id: sessionId,
    user_id: user?.userId || '',
  };
}

export function track(eventName: string, props?: Record<string, any>) {
  try {
    wx.reportEvent(eventName, {
      ...getCommonProps(),
      ...props,
    });
  } catch (e) {
    // 埋点失败不影响主流程
    console.warn('[Track] 埋点失败:', eventName, e);
  }
}
```

## 五、体重数据脱敏方案

```typescript
// 体重上报只上报 10kg 档位区间
function getWeightRange(weight: number): string {
  const lower = Math.floor(weight / 10) * 10;
  return `${lower}-${lower + 10}`;
}

// 使用
track('checkin_weight_input', {
  weight_range: getWeightRange(weight), // e.g. '60-70'，不上报 65.3
});
```

## 六、埋点验收

上线前需与数据同学确认：
- [ ] `wx.reportEvent` 数据在后台可正常接收
- [ ] 公共属性字段类型正确
- [ ] 体重字段已脱敏（无精确体重值）
- [ ] `user_id` 能与业务库关联
- [ ] 核心转化漏斗可搭建（`login_success` → `home_create_pk_tap` → `pk_create_success` → `checkin_submit_success`）
