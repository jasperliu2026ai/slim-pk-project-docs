## 一、Git 工作流

### 分支规范

| 分支 | 保护 | 说明 |
|------|------|------|
| `main` | ✅ 禁止直接 push | 线上稳定版本，只允许从 `dev` merge |
| `dev` | ✅ 禁止 force push | 开发集成分支，所有 feat/fix 向此合并 |
| `feat/xxx` | — | 功能开发，从 `dev` 拉出 |
| `fix/xxx` | — | Bug 修复，从 `dev` 拉出 |
| `hotfix/xxx` | — | 紧急修复，从 `main` 拉出，修完同时 merge 回 `main` 和 `dev` |

### Commit Message 规范（Conventional Commits）

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

**type 类型**

| type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `perf` | 性能优化 |
| `refactor` | 重构（不影响功能）|
| `style` | 样式调整（不影响逻辑）|
| `test` | 测试相关 |
| `docs` | 文档更新 |
| `chore` | 构建/工具配置变更 |
| `revert` | 回滚 |

**示例**

```
feat(checkin): 添加打卡图水印合成功能

- 使用 Canvas 2D 在拍照后合成时间戳+昵称水印
- 合成后原图不保留，只上传合成图到 OSS

feat(pk-detail): 加入 PK 确认弹窗

fix(login): 修复 token 过期后未清空本地 storage 的问题

perf(home): 排行榜列表改为分页加载，减少首屏 setData payload
```

### 2.3 命名规范

| 场景 | 规范 | 示例 |
|------|------|------|
| 变量/函数 | camelCase | `pkList`, `loadMoreData()` |
| 类/接口 | PascalCase | `PKItem`, `UserStore` |
| 常量 | UPPER_SNAKE | `MAX_MEMBERS` |
| 组件文件夹 | kebab-case | `pk-card/`, `score-ring/` |
| WXML class | BEM | `.pk-card__header--active` |
| 页面路径 | kebab-case | `pages/pk-detail/index` |
| 事件处理函数 | `handle` 前缀 | `handleCheckinSubmit` |
| API 函数 | 动词+名词 | `getPkList()`, `submitCheckin()` |

### 2.4 TypeScript 要求

- 禁止 `any`（必要时用 `unknown` + 类型守卫）
- 接口响应体必须有对应 TypeScript 接口定义（在 `types/api.d.ts`）
- 非空断言（`!`）使用前必须确认逻辑上不可能为空，加注释说明

## 四、CI/CD 配置

### 4.1 自动上传体验版（GitHub Actions）

```yaml
# .github/workflows/upload-weapp.yml
name: Upload WeApp Preview

on:
  push:
    branches: [dev]

jobs:
  upload:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Install miniprogram-ci
        run: npm install -g miniprogram-ci

      - name: Build npm
        run: |
          cd miniprogram
          npm ci

      - name: Upload to WeApp
        env:
          WEAPP_APPID: ${{ secrets.WEAPP_APPID }}
          WEAPP_PRIVATE_KEY: ${{ secrets.WEAPP_PRIVATE_KEY }}
        run: |
          echo "$WEAPP_PRIVATE_KEY" > private.key
          miniprogram-ci upload \
            --pp . \
            --pkp ./private.key \
            --appid $WEAPP_APPID \
            --uv $(date +%Y%m%d%H%M) \
            --ud "Auto upload from dev branch" \
            -r 1
```

> **密钥管理**：`WEAPP_APPID` 和 `WEAPP_PRIVATE_KEY`（微信开发者后台下载的 `.key` 文件内容）存储在 GitHub Secrets，不提交到仓库。

### 4.2 代码检查（PR 阶段）

```yaml
# .github/workflows/lint.yml
name: Lint Check

on:
  pull_request:
    branches: [dev, main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm ci
      - run: npm run lint
      - run: npm run type-check
```

## 五、微信开发者工具使用规范

- **版本**：团队统一使用同一个稳定版本（当前：1.06.2405020），避免版本差异导致编译结果不同
- **基础库**：调试时切到最低支持版本（2.30.0）测试兼容性
- **代码保护**：上传时开启「代码保护」
- **Source Map**：开发/体验版开启，正式版关闭

## 六、安全规范

| 规则 | 说明 |
|------|------|
| ❌ 不硬编码密钥 | AppSecret、OSS 密钥等不能出现在代码里 |
| ❌ 不 git push --force main/dev | 保护主分支历史 |
| ✅ 环境变量用 CI Secrets | APPID / 私钥走 GitHub Secrets |
| ✅ 接口域名走配置 | 不 hardcode，区分 dev/prod |
| ✅ 依赖升级要看 changelog | 升级前本地跑通再合入 |
| ✅ 新引入 npm 包需评审 | 在 PR 描述中说明：包名/版本/用途/License |
