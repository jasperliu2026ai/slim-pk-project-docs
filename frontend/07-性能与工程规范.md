## 一、性能关注点与措施

### 1.1 setData 优化

setData 是小程序性能的核心瓶颈，每次调用都涉及逻辑层→渲染层的 JSON 序列化通信。

**规则：**

```typescript
// ❌ 禁止：循环里多次 setData
for (const item of items) {
  this.setData({ [`list[${i}]`]: item }); // 每次都触发通信
}

// ✅ 正确：合并成一次 setData
const patch: Record<string, unknown> = {};
items.forEach((item, i) => {
  patch[`list[${i}]`] = item;
});
this.setData(patch);
```

**关键规则：**
- 同一帧内的多次 setData 必须合并
- 不把非渲染用数据（如临时计算变量）放到 data 里
- 排行榜列表：只 diff 变化的条目，不全量替换

### 1.2 长列表（排行榜）

L1 人数 < 20，无需虚拟滚动。L2 人数超过 50 时评估引入 `recycle-view`。

当前方案：
- 使用 `wx:key` 绑定 `userId`，避免全量重渲
- 头像图片设置 `lazy-load="{{true}}"`

### 1.3 图片处理

```typescript
// 拍照后压缩（上传前必须压缩）
await wx.compressImage({
  src: tempFilePath,
  quality: 80,       // 质量 80%
  compressedWidth: 1280,  // 最大宽度 1280px（约等于 720P）
});
// 目标：压缩后 < 300KB
```

- 网络图片（头像、分享封面）统一设置 `webp` 格式（COS 支持图片处理参数）
- 战报海报 Canvas 输出 quality 设 0.92（jpg），文件大小约 200-400KB

### 1.4 Canvas 2D 海报

```typescript
// 使用离屏 Canvas，不阻塞 UI 线程
// app.json 中配置：
{
  "rendererOptions": {
    "skyline": { "defaultDisplayBlock": false }
  }
}

// 组件 json：
{
  "componentFramework": "glass-easel"  // 可选，提升渲染性能
}
```

### 1.5 主包体积控制

| 措施 | 预期收益 |
|------|----------|
| Vant Weapp 按需引入 | 节省 ~300KB |
| ECharts 定制版（仅 line chart） | 节省 ~600KB |
| lottie JSON 放 packageResult | 不影响主包 |
| 图片走 CDN URL（不打包） | 节省 ~200KB |
| TypeScript 编译后去除注释 | 节省 ~50KB |

## 三、自测 Checklist

每个功能开发完成后，提 PR 前必须完成以下自测：

### 正常流程
- [ ] 核心功能主流程走通（快乐路径）
- [ ] 数据展示正确（比对接口返回值）
- [ ] 表单校验生效

### 异常流程
- [ ] 网络断开时的行为（toast + 不崩溃）
- [ ] 接口返回错误码时的处理
- [ ] 空数据时的空态展示

### 边界值
- [ ] 体重：30.0kg、300.0kg、29.9kg（拒绝）、300.1kg（拒绝）
- [ ] 昵称脱敏：1个字、2个字、10个字
- [ ] 排行榜：1人、20人（L1 上限）
- [ ] PK 时长：7天（最短）、90天（最长）

### 体验
- [ ] 页面加载有 loading 状态
- [ ] 按钮有点击态（不能是白板点击）
- [ ] 防重复提交（打卡按钮点击后 disabled 直到接口返回）

### 设备兼容
- [ ] 低端 Android（OPPO A 系列、荣耀 Play 等，内存 3GB 以下）
- [ ] iOS（iPhone 11 及以上）

## 四、WXSS 规范

```css
/* 使用 rpx 做响应式，不用 px（除 1px 边框） */
.container {
  padding: 32rpx;         /* ✅ */
  padding: 16px;          /* ❌ */

/* 1px 分割线用 px */
.divider {
  border-top: 1px solid var(--color-bg);

/* 避免通配符 * 选择器 */
```

## 五、网络异常兜底

```typescript
// 打卡数据本地暂存（仅当日有效）
interface PendingCheckin {
  pkId: string;
  weight: number;
  photoPath: string;
  date: string; // YYYY-MM-DD
}

function savePendingCheckin(data: PendingCheckin) {
  const today = formatDate(new Date());
  if (data.date !== today) return; // 只暂存当日数据
  wx.setStorageSync('pendingCheckin', data);
}

// App 每次 onShow 检查是否有待上传的打卡
async function checkPendingCheckin() {
  const pending = wx.getStorageSync('pendingCheckin') as PendingCheckin | null;
  const today = formatDate(new Date());
  if (pending && pending.date === today) {
    await retryCheckin(pending);
    wx.removeStorageSync('pendingCheckin');
  } else if (pending) {
    // 已过期，丢弃（体积小数据，不上传）
    wx.removeStorageSync('pendingCheckin');
  }
}
```

> ⚠️ 注意：暂存的 `photoPath` 是临时文件路径，小程序重启后可能失效。如果失效则提示用户「上次打卡照片已失效，请重新拍照」。
