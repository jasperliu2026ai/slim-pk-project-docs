# 01 技术选型与架构

> 版本：v1.0 | 更新：2026-07-14

---

## 一、技术选型总表

| 分类 | 选型 | 版本建议 | 理由 |
|------|------|----------|------|
| 小程序框架 | **微信小程序原生** WXML / WXSS / TypeScript | 基础库 ≥ 3.0 | 零跨端冗余；wx.login / 分享 / 订阅消息等核心 API 直接用，无封装损耗；18 天 MVP 降低技术复杂度 |
| 组件库 | **Vant Weapp** | ^1.12.x | 社区活跃，组件覆盖率高；CSS 变量支持主题定制（活力橙 #FF6B2B / 竞技红 #E53935），不被微信绿统治调性；按需引入，主包体积可控 |
| 状态管理 | **MobX-miniprogram** | ^1.4.x | 排行榜、打卡状态需跨页共享；比手写全局 store 更可维护；完整 TypeScript 类型支持 |
| 图表 | **ECharts for 小程序** | ^5.x | 体重趋势折线图、积分趋势；官方适配方案，Canvas 渲染 |
| 动效 | **lottie-miniprogram** | ^1.x | 打卡成功 / PK 胜利关键节点动效；JSON 驱动，设计师直接输出，不手写 CSS 动画 |
| 海报生成 | Canvas 2D API | 原生 | 结算战报图片；离屏 Canvas 不阻塞 UI 线程 |
| 代码规范 | ESLint + Prettier + stylelint | — | 统一团队代码风格 |
| 提交规范 | Conventional Commits | — | `feat:` / `fix:` / `chore:` 前缀，commit 自解释 |

### 为什么不选 Taro / uni-app

- 当前无跨端需求（仅微信小程序），引入跨端框架带来额外体积和运行时差异
- Taro 的 setData diff 层与原生行为存在微差，低端机调试成本高
- 18 天 MVP，降低技术复杂度优先

## 三、分包策略

### 3.1 分包配置（app.json 摘录）

```json
{
  "pages": [
    "pages/login/index",
    "pages/index/index"
  ],
  "subpackages": [
    {
      "root": "packagePK",
      "name": "pk",
      "pages": [
        "pages/create/index",
        "pages/join/index",
        "pages/checkin/index",
        "pages/ranking/index"
      ]
    },
    {
      "root": "packageResult",
      "name": "result",
      "pages": [
        "pages/settle/index",
        "pages/poster/index"
      ]
    }
  ],
  "preloadRule": {
    "pages/index/index": {
      "network": "all",
      "packages": ["pk"]
    },
    "packagePK/pages/ranking/index": {
      "network": "wifi",
      "packages": ["result"]
    }
  }
}
```

### 3.2 体积目标

| 包 | 目标体积 | 主要内容 |
|----|----------|----------|
| 主包 | < 1.5 MB | TabBar 页、登录页、全局 store、utils |
| packagePK | < 800 KB | PK 流程页面 + 相机组件 |
| packageResult | < 600 KB | 结算页 + Canvas 海报 + lottie 动效 JSON |

### 3.3 主包体积控制原则

- Vant Weapp **按需引入**，只打包用到的组件
- ECharts 使用**定制版**（只含 line chart），不引入完整包
- lottie JSON 动效文件放入 `packageResult`，不放主包
- 图片资源统一走 COS CDN URL，不打入包内

## 五、主题色定义

```css
/* app.wxss — 全局 CSS 变量 */
page {
  /* 主色：活力橙 */
  --color-primary: #FF6B2B;
  --color-primary-light: #FF8C5C;
  --color-primary-dark: #E55A1F;

  /* 辅色：竞技红（强调 / 排名第一） */
  --color-accent: #E53935;

  /* 中性色 */
  --color-text-primary: #1A1A1A;
  --color-text-secondary: #666666;
  --color-text-disabled: #BBBBBB;
  --color-bg: #F5F5F5;
  --color-white: #FFFFFF;

  /* 语义色 */
  --color-success: #07C160;
  --color-warning: #FA8C16;
  --color-error: #F5222D;
}
```

> Vant Weapp 主题覆盖：在各页面 .json 中通过 `"styleIsolation": "shared"` 或 CSS 变量传递覆盖 Vant 默认的微信绿（`#07C160`）。
