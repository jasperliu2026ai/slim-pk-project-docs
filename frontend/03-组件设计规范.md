## 一、组件分层原则

```
组件层级
├── 基础组件层（Vant Weapp）     → 直接使用，不修改源码，通过 CSS 变量定制样式
├── 通用业务组件层（components/）→ 有业务语义，但多页面复用，放主包
├── 分包私有组件层               → 只在某个分包内用，放对应分包 components/
└── 页面级局部组件                → 只在一个页面用，直接写在页面目录下
```

## 二、公共业务组件详细说明

### 2.1 `nav-bar` — 自定义导航栏

**使用背景**：微信原生导航栏不支持自定义色彩，PK 详情页需要展示沉浸式背景。

**属性（Properties）**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `title` | String | `''` | 标题文字 |
| `showBack` | Boolean | `true` | 是否显示返回按钮 |
| `bgColor` | String | `#FFFFFF` | 背景色（支持透明 `transparent`） |
| `textColor` | String | `#1A1A1A` | 文字和图标颜色 |
| `fixed` | Boolean | `true` | 是否固定顶部 |

**事件（Events）**

| 事件 | 说明 |
|------|------|
| `back` | 点击返回按钮触发 |

**使用示例**
```xml
<nav-bar
  title="PK 详情"
  bg-color="transparent"
  text-color="#FFFFFF"
  bind:back="handleBack"
/>
```

**实现要点**
- 高度 = 状态栏高度（`wx.getSystemInfoSync().statusBarHeight`）+ 44px
- 返回逻辑默认 `wx.navigateBack()`，可通过 back 事件覆盖

### 2.2 `pk-card` — PK 卡片

**使用场景**：首页 PK 列表、搜索结果

**属性**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pk` | Object | ✅ | PK 数据对象（见 `PKItem` 类型） |
| `mode` | String | — | `list`（默认）/ `compact` |

**PKItem 类型**
```typescript
interface PKItem {
  id: string;
  name: string;
  status: 'pending' | 'ongoing' | 'finished';
  startDate: string;        // YYYY-MM-DD
  endDate: string;          // YYYY-MM-DD
  totalDays: number;
  memberCount: number;
  maxMembers: number;
  myCheckinToday: boolean;  // 今日是否已打卡
  myRank?: number;          // 我的排名（ongoing/finished 时有）
  myScore?: number;         // 我的积分
  coverImageUrl?: string;   // 封面图（可选）
}
```

**事件**

| 事件 | 说明 |
|------|------|
| `tap` | 点击卡片，detail 携带 `pkId` |

### 2.4 `score-ring` — 圆环进度组件

**使用场景**：排行榜条目的完成率展示

**属性**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `percent` | Number | `0` | 完成率 0-100 |
| `score` | Number | `0` | 中间显示的积分数值 |
| `size` | Number | `80` | 圆环直径（rpx） |
| `strokeWidth` | Number | `6` | 线宽（rpx） |
| `color` | String | `#FF6B35` | 进度条颜色 |

**实现**：使用 Canvas 2D 绘制，不依赖第三方。

### 2.5 `countdown-banner` — 倒计时横幅

**使用场景**：PK 详情页顶部，显示距结束还有 X 天 X 小时

**属性**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `endDate` | String | ✅ | 结束日期 `YYYY-MM-DD` |
| `status` | String | ✅ | `pending` / `ongoing` / `finished` |

**行为**
- `ongoing`：显示「还剩 X 天 X 小时」，每分钟更新一次（`setInterval`）
- `pending`：显示「X 天后开始」
- `finished`：显示「已结束」

**生命周期注意**：页面 `onHide` 时清除 interval，`onShow` 时重新启动。

## 三、组件开发规范

### 3.1 文件结构
```
components/pk-card/
├── index.ts        # 组件逻辑
├── index.wxml      # 模板
├── index.wxss      # 样式
└── index.json      # 组件配置
```

### 3.2 组件 JSON 规范
```json
{
  "component": true,
  "usingComponents": {
    "van-image": "@vant/weapp/image/index"
  }
}
```

### 3.3 样式规范
- 组件样式全部用 CSS 变量（来自 `variables.wxss`），禁止硬编码颜色
- 组件样式类名用 `BEM` 命名：`.pk-card__header--active`
- 支持外部样式类（`externalClasses`）时需在 JSON 中声明

### 3.4 事件规范
```typescript
// 触发事件统一用 triggerEvent，detail 结构明确定义
this.triggerEvent('tap', { pkId: this.data.pk.id });
```

### 3.5 组件性能注意
- 列表类组件：避免在组件内部做 `setData`，由父页面统一管理
- 图片组件：统一使用 `<image lazy-load>` + `mode="aspectFill"`
- 复杂计算移到 `computed`（使用 `miniprogram-computed`）
