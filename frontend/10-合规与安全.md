## 一、健康数据合规

减肥 PK 场景涉及体重等**敏感个人信息**（《个人信息保护法》第 28 条明确列出健康信息为敏感个人信息），前端必须按最高安全级别处理。

### 1.1 数据分类

| 数据 | 类型 | 处理要求 |
|------|------|----------|
| 体重（初始 + 每日）| 敏感个人信息 | 单独同意；不展示绝对值；不本地持久化；后端加密存储 |
| 打卡照片 | 敏感个人信息 | 单独同意；上传后丢弃本地临时路径；不公开展示给其他成员 |
| 微信昵称 + 头像 | 一般个人信息 | 可正常存储和展示（排行榜） |
| openid | 标识符 | 仅在后端使用，不传递到前端页面 |

### 1.2 明确不采集的数据

- 真实姓名 / 身份证 / 手机号
- 精确 GPS 位置（照片水印只记录时间，不记录地理坐标）
- 饮食详情 / 食物图片识别结果
- 设备硬件信息（不做设备指纹）

## 二、隐私授权弹框

### 必须实现的弹框（缺一不可上线）

#### 2.1 「敏感个人信息处理说明」单独同意弹框

**触发时机**：首次登录时（`isNewUser === true`）

**必须包含的内容**：
```
标题：使用须知

正文：
「减肥 PK」需要收集以下信息以提供服务：
• 体重数据（初始体重 + 每日打卡体重）
• 打卡照片（存证用，不对其他成员公开）

收集目的：计算减重积分、生成排行榜和战报
保存期限：PK 结束后 90 天，您可随时在「我的」中删除所有数据
您可以拒绝提供，但将无法使用打卡和积分功能

[同意并继续]   [不同意，仅查看]
```

**实现要求**：
- 不得使用「好的」「确认」等模糊按钮文字
- 「同意」按钮明显大于「不同意」，但不得使用 dark pattern（不允许把「不同意」设为灰色不可点）
- 同意状态持久化到 `wx.setStorageSync('privacyConsented', true)`
- 用户后续可在「我的 → 设置 → 撤回同意」中撤回

#### 2.2 免责声明

**展示位置**：登录页底部 + 「我的」页面底部

```
免责声明：本工具仅供个人健身记录，不替代医疗健康建议。
若有健康疑虑，请咨询专业医生。
```

#### 2.3 《隐私政策》与《用户协议》

- 页面路径：`pages/privacy/index` + `pages/agreement/index`
- 登录按钮上方必须有链接：「登录即代表同意《用户协议》和《隐私政策》」
- 文案由菲菲【产品】提供

## 三、极端数据拦截

### 3.1 体重范围校验

```typescript
const WEIGHT_RULES = {
  min: 30.0,       // < 30kg 拒绝提交
  max: 300.0,      // > 300kg 拒绝提交
  warnLowBMI: true, // 计算 BMI < 18.5 时弱提示
};

function checkHealthWarning(weight: number, height?: number): string | null {
  if (height) {
    const bmi = weight / Math.pow(height / 100, 2);
    if (bmi < 18.5) {
      return '您目前 BMI 低于 18.5（偏瘦），请在医生指导下制定减重计划。';
    }
  }
  return null;
}
```

### 3.2 减重目标校验（创建 PK 时）

| 条件 | 处理 |
|------|------|
| 无 | MVP 阶段不校验具体减重目标（PK 只设时间，不设目标体重） |
| 未来可能加「目标体重」字段 | 届时需校验：目标体重不低于 BMI 18.5 对应值 |

## 四、前端安全红线

### 4.1 代码安全

```
❌ 不在前端代码中硬编码：
   - 后端 API BaseURL（走 config 文件）
   - 任何密钥 / secret
   - 微信 AppSecret（只能在后端使用）

✅ project.private.config.json 必须在 .gitignore 中
✅ 所有接口走 HTTPS（生产环境）
```

### 4.2 数据安全

```
❌ 体重数据不写入 wx.setStorageSync
❌ 照片 tempFilePath 上传成功后立即丢弃本地引用
❌ 不在 console.log 中打印体重 / 用户 token
✅ 排行榜只展示减重% 和积分，不展示绝对体重
✅ 打卡照片不公开展示给其他 PK 成员
```

### 4.3 XSS 防护

```typescript
// 不使用 wx:innerHTML 展示用户输入内容
// 昵称 / PK 名称统一用 {{}} 插值（WXML 自动 HTML encode）
// 不用 setContents / innerHTML 插入 HTML
```

### 4.4 防重复提交

```typescript
// 打卡按钮点击后立即 disabled，接口成功/失败后恢复
Page({
  data: { submitting: false },

  async handleCheckin() {
    if (this.data.submitting) return;
    this.setData({ submitting: true });
    try {
      await submitCheckin();
    } finally {
      this.setData({ submitting: false });
    }
  }
});
```

## 六、微信审核注意事项

历史上涉及健康数据的小程序审核时常被拒，以下是预防措施：

1. **类目选择**：选「运动健康 → 体育/健身」或「生活服务 → 其他」，不选涉及医疗的类目
2. **文案措辞**：
   - ❌ 不出现：减肥药、燃脂、瘦身、医疗、诊断、治疗
   - ✅ 使用：运动记录、健康打卡、体重管理、健身挑战
3. **免责声明**：必须在登录页和「关于」页可见
4. **截图说明**：审核时提交截图需包含隐私弹框的完整界面

如果审核被拒，常见原因和修改方案：
- 「涉及健康类目需相关资质」→ 调整类目 + 措辞，强调「记录工具」而非「健康管理」
- 「隐私声明不完整」→ 补充单独同意弹框截图作为说明
