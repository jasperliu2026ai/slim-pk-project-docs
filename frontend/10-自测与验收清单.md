## 三、兼容性测试清单

测试设备要求：**至少 1 台低端 Android + 1 台 iOS**，提测前必须完成。

| 设备 | 系统 | 微信版本 | 基础库 | 测试状态 |
|------|------|----------|--------|----------|
| 低端 Android（骁龙 665 以下） | Android 9+ | ≥ 8.0 | 2.30.0 | ⬜ |
| 中端 Android | Android 12+ | 最新 | 最新 | ⬜ |
| iPhone 8 | iOS 15 | ≥ 8.0 | 2.30.0 | ⬜ |
| iPhone 14 | iOS 16+ | 最新 | 最新 | ⬜ |

**关键兼容性检查项**

- [ ] Canvas 2D 水印合成（低端机是否流畅）
- [ ] Lottie 动效（低端机是否正常播放）
- [ ] 分包加载（首次进入分包页面是否有明显延迟，是否有 loading 提示）
- [ ] 自定义导航栏高度（不同机型状态栏高度是否正确）

## 四、性能验收基线

| 指标 | 要求 | 验证方式 |
|------|------|----------|
| 首屏渲染 | ≤ 2s（4G） | 微信性能面板 |
| 打卡图水印合成 | ≤ 3s | 实测计时 |
| 战报海报生成 | ≤ 5s | 实测计时 |
| 主包体积 | ≤ 1.5MB | 开发者工具「代码依赖分析」|
| setData 最大 payload | ≤ 256KB | 代码审查 |
| 页面列表滚动 | 无明显卡顿 | 手机实机测试 |

## 五、提测 Checklist

提交给测试之前，前端需确认以下全部完成：

- [ ] 本文档所有自测用例状态已全部打 ✅
- [ ] 真机测试至少 2 台（1 Android + 1 iOS）
- [ ] 主包体积 ≤ 1.5MB
- [ ] ESLint 无 error（warning 可接受）
- [ ] TypeScript 编译无报错（`npm run type-check`）
- [ ] 所有接口已联调通过（或有可用 Mock 说明）
- [ ] 埋点已实现，数据同学确认可接收
- [ ] 提测说明文档已写（功能点 + 测试重点 + 已知问题 + 测试账号）
- [ ] 体验版二维码已生成并发给测试同学
