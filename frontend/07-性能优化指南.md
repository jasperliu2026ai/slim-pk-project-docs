## 一、性能基线目标

| 指标 | 目标值 | 测量工具 |
|------|--------|----------|
| 首屏渲染时间（冷启动） | ≤ 2s | 微信小程序性能面板 |
| 主包体积 | ≤ 1.5MB | 微信开发者工具 |
| 总体积（含分包） | ≤ 6MB | — |
| setData 单次 payload | ≤ 256KB | 代码评审 |
| 列表滚动帧率 | ≥ 55fps | 小程序性能面板 |

## 二、setData 优化

### 2.1 合并 setData

```typescript
// ❌ 多次分开调用
this.setData({ loading: true });
this.setData({ title: '加载中' });
this.setData({ list: [] });

// ✅ 合并一次调用
this.setData({
  loading: true,
  title: '加载中',
  list: [],
});
```

### 2.2 局部更新

```typescript
// ❌ 整个列表替换
this.setData({ pkList: newList });

// ✅ 只更新变化的项（单个字段）
this.setData({
  [`pkList[${index}].myCheckinToday`]: true,
});
```

### 2.3 减少 payload 大小

```typescript
// 列表接口返回字段裁剪：只保存展示必需的字段
interface PKItemLite {
  id: string;
  name: string;
  status: string;
  myCheckinToday: boolean;
  myScore: number;
  myRank: number;
  memberCount: number;
  endDate: string;
  // 不存头像列表（列表页不展示）、不存完整成员信息
}
```

## 四、图片优化

### 4.1 上传前压缩

```typescript
// 打卡图上传前压缩
async function compressAndUpload(filePath: string): Promise<string> {
  // 压缩图片（质量 80%，最大宽度 1080）
  const compressed = await new Promise<string>((resolve, reject) =>
    wx.compressImage({
      src: filePath,
      quality: 80,
      success: (res) => resolve(res.tempFilePath),
      fail: reject,
    })
  );

  // 上传到 OSS
  return uploadToOSS(compressed);
}
```

### 4.2 CDN 图片尺寸参数

后端 OSS URL 支持拼接尺寸参数（阿里云 OSS 格式）：

```typescript
function getThumbUrl(url: string, width: number): string {
  if (!url) return '';
  // 阿里云 OSS 图片处理参数
  return `${url}?x-oss-process=image/resize,w_${width}/format,webp`;
}

// 使用
const thumbUrl = getThumbUrl(checkinImageUrl, 240); // 打卡图缩略图
const avatarUrl = getThumbUrl(rawAvatarUrl, 120);   // 头像 120px
```

### 4.3 本地静态资源管理

- **图标**：iconfont 字体文件（< 50KB），放主包 assets/icons/
- **背景图**：使用 CSS 渐变代替图片（主背景）
- **Lottie 动效**：单个 JSON ≤ 100KB，放对应分包 assets/lottie/
- **大图**：走 CDN，不放本地

## 五、启动性能优化

### 5.1 分包预下载

```json

{

```

进入首页后就开始下载 pk-core 分包，用户点击 PK 卡片时分包已准备好。

### 5.2 登录并行化

```typescript
// app.ts onLaunch — 不等 login 完成再渲染页面
App({
  onLaunch() {
    // 并行执行，不阻塞页面渲染
    this.initAuth();
  },

  async initAuth() {
    const token = wx.getStorageSync('accessToken');
    if (token && isTokenValid()) {
      userStore.accessToken = token;
      // 后台静默刷新 token（不 await）
      refreshTokenIfNeeded();
    } else {
      // 跳转登录页（index 页面里检查登录态）
    }
  },
});
```

## 六、主包体积治理

### 6.1 按需引入 Vant Weapp

```json
// 只在用到的页面/组件 JSON 中引入对应组件
{
  "usingComponents": {
    "van-button": "@vant/weapp/button/index",
    "van-field": "@vant/weapp/field/index"
  }
}
```

不在 `app.json` 全局引入所有 Vant 组件。

### 6.2 体积监控

每次发布体验版前，在微信开发者工具「代码依赖分析」中检查：
- 主包 > 1.5MB → 需排查并移出内容到分包
- 某个分包 > 3MB → 需细拆
- 重复引入的模块 → 提取到主包 utils 共享

## 七、内存管理

### 7.1 页面销毁时清理

```typescript
Page({
  data: {
    timer: null as ReturnType<typeof setInterval> | null,
  },

  onLoad() {
    this.data.timer = setInterval(() => this.updateCountdown(), 60000);
  },

  onUnload() {
    // 页面销毁时清除定时器
    if (this.data.timer) {
      clearInterval(this.data.timer);
    }
  },

  onHide() {
    // 页面隐藏时暂停，节省资源
    if (this.data.timer) {
      clearInterval(this.data.timer);
    }
  },

  onShow() {
    // 页面重新显示时恢复
    this.data.timer = setInterval(() => this.updateCountdown(), 60000);
  },
});
```

### 7.2 Canvas 资源释放

使用完水印/海报 Canvas 后，不复用时主动断开引用：
```typescript
// 生成完成后
canvas = null;
ctx = null;
```
