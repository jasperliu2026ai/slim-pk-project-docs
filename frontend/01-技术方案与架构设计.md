## 一、技术选型

| 项目 | 选型 | 版本 | 选型理由 |
|------|------|------|----------|
| 开发框架 | **微信小程序原生** | 基础库 ≥ 3.3.0 | 无跨端依赖；wx.login / share / payment 直调；审核最稳定 |
| 开发语言 | **TypeScript** | 5.x | 类型安全；接口字段对齐；IDE 提示友好 |
| UI 组件库 | **Vant Weapp** | 1.10.x | 社区活跃、组件完整、支持 CSS 变量主题覆盖 |
| 状态管理 | **MobX-miniprogram** | 1.x | 跨页响应式共享；打卡状态/排行榜实时刷新 |
| 图表 | **ECharts for 小程序** | 5.x | 体重趋势曲线；Canvas 渲染性能好 |
| 动效 | **lottie-miniprogram** | 1.x | 打卡成功/PK 胜利关键动效；文件体积可控 |
| 图标 | **iconfont** 自维护项目 | — | 通过 van-icon 的 class-prefix 接入 |
| 分享海报 | **Canvas 2D API** | — | 生成战报图片；一键保存/分享 |
| HTTP 封装 | 自研轻量 `request.ts` | — | 统一 token / loading / 错误拦截 |

### 未引入的框架（及原因）

| 框架 | 未引入原因 |
|------|------------|
| Taro / uni-app | 本项目只做微信小程序，无跨端需求；原生开发调试更直接 |
| React / Vue3 Web | 不适用 |
| 小程序云开发 | 后端团队已有自有服务，无需引入 |

## 三、分包策略

> **目标：主包体积严控在 1.5MB 以内，总体积 ≤ 6MB**

```
主包（目标 ≤ 1.5MB）
├── 登录/授权页     (pages/login)
├── 首页 PK 列表    (pages/index)
├── 全局组件        (components/)
├── 全局状态        (store/)
└── 工具函数        (utils/)

分包 A — pk-core（懒加载，预下载：进入首页后开始）
├── 发起 PK         (subpackages/pk-core/pages/pk-create)
├── PK 详情         (subpackages/pk-core/pages/pk-detail)
├── 每日打卡        (subpackages/pk-core/pages/checkin)
└── 排行榜          (subpackages/pk-core/pages/leaderboard)

分包 B — settlement（懒加载）
└── 结算页 + 战报   (subpackages/settlement/pages/index)
```

### app.json 分包配置示例

```json
{
  "subpackages": [
    {
      "root": "subpackages/pk-core",
      "name": "pk-core",
      "pages": [
        "pages/pk-create/index",
        "pages/pk-detail/index",
        "pages/checkin/index",
        "pages/leaderboard/index"
      ]
    },
    {
      "root": "subpackages/settlement",
      "name": "settlement",
      "pages": [
        "pages/index/index"
      ]
    }
  ],
  "preloadRule": {
    "pages/index/index": {
      "network": "all",
      "packages": ["pk-core"]
    }
  }
}
```
