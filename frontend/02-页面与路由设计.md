## 二、路由规则

### 2.1 页面路径映射

| 页面 | 路径 | 分包 | 路由方式 |
|------|------|------|----------|
| 登录/授权 | `pages/login/index` | 主包 | `redirectTo` / `reLaunch` |
| 首页 PK 列表 | `pages/index/index` | 主包 | TabBar `switchTab` |
| 发起 PK | `subpackages/pk-core/pages/pk-create/index` | 分包A | `navigateTo` |
| PK 详情 | `subpackages/pk-core/pages/pk-detail/index` | 分包A | `navigateTo` |
| 每日打卡 | `subpackages/pk-core/pages/checkin/index` | 分包A | `navigateTo` |
| 排行榜 | `subpackages/pk-core/pages/leaderboard/index` | 分包A | `navigateTo` |
| 结算 | `subpackages/settlement/pages/index/index` | 分包B | `navigateTo` |

### 2.2 路由参数规范

所有 `navigateTo` 跳转携带参数通过 `url query` 传递，复杂参数通过 EventChannel 或 store：

```typescript
// 跳转 PK 详情
wx.navigateTo({
  url: `/subpackages/pk-core/pages/pk-detail/index?pkId=${pkId}`,
});

// 跳转结算页（多参数用 store 传）
pkStore.setCurrentPk(pkData);
wx.navigateTo({
  url: '/subpackages/settlement/pages/index/index',
});
```

### 2.3 页面栈管理原则

- **最大栈深度**：控制在 5 层以内（微信上限 10 层）
- 打卡完成后用 `navigateBack` 返回 PK 详情，不再 push 新页面
- 结算页是终态，从结算分享回来后 `reLaunch` 到首页
- 从分享卡片/二维码扫码进入时，走 `onLaunch` 解析 `scene` 参数，`redirectTo` 登录页后再路由到对应 PK

### 2.4 扫码/分享进入路由

```typescript
// app.ts onLaunch
App({
  onLaunch(options) {
    // 扫描邀请二维码进入
    if (options.scene === 1011 || options.scene === 1047) {
      const { pkId } = decodeShareParams(options.query);
      // 先登录，登录完成后跳转到 pk-detail
      navigateAfterLogin(`/subpackages/pk-core/pages/pk-detail/index?pkId=${pkId}&fromInvite=1`);
    }
  }
});
```

## 三、各页面详细说明

### 3.1 登录/授权页 `pages/login/index`

**触发时机**
- 未登录用户进入任何需要鉴权的页面时，统一 redirect 到此
- 从分享/扫码冷启动时

**功能点**
1. 调用 `wx.login()` 获取 code，与后端换 accessToken
2. 首次登录展示**隐私政策弹框**（含体重敏感数据说明）
3. 登录成功后 redirect 回目标页（通过 `redirectUrl` 参数传递）

**页面状态**
```typescript
data: {
  loading: boolean;       // 登录中
  privacyVisible: boolean; // 隐私弹框
  error: string;          // 错误提示
}
```

---

### 3.2 首页 PK 列表 `pages/index/index`

**功能点**
1. 拉取当前用户参与的所有 PK（按状态分 tab：进行中 / 待开始 / 已结束）
2. 每个 PK 用 `pk-card` 组件展示
3. 右下角浮动「发起 PK」按钮
4. 下拉刷新 + 上拉加载更多（分页）

**页面状态**
```typescript
data: {
  activeTab: 'ongoing' | 'pending' | 'finished';
  pkList: PKItem[];
  loading: boolean;
  hasMore: boolean;
  page: number;
}
```

**生命周期**
- `onLoad`：初始化，拉取第一页列表
- `onShow`：每次显示刷新列表（打卡/创建后回来要刷新）
- `onPullDownRefresh`：下拉刷新
- `onReachBottom`：加载更多

---

### 3.3 发起 PK `subpackages/pk-core/pages/pk-create/index`

**功能点**
1. 表单：PK 名称（必填）、目标体重（必填，带校验）、周期（7/14/21/30 天选择器）、人数上限（2~10）
2. 创建成功后展示邀请弹窗（二维码 + 复制链接）
3. 输入校验：BMI < 18.5 时弹健康提示，不阻断创建

**校验规则**
```typescript
// 目标体重校验
function validateTargetWeight(currentWeight: number, targetWeight: number): ValidationResult {
  if (targetWeight <= 0 || targetWeight > 500) {
    return { valid: false, msg: '请输入合理的目标体重' };
  }
  if (targetWeight >= currentWeight) {
    return { valid: false, msg: '目标体重应低于当前体重' };
  }
  const weeklyLoss = (currentWeight - targetWeight) / (period / 7);
  if (weeklyLoss > 1) {
    // 弱提示，不阻断
    return { valid: true, warning: '目标较激进，建议每周减重 ≤ 1kg' };
  }
  return { valid: true };
}
```

---

### 3.4 PK 详情 `subpackages/pk-core/pages/pk-detail/index`

**功能点**
1. PK 头部：名称、倒计时横幅（`countdown-banner` 组件）、当前状态
2. 参与者列表：头像 + 昵称 + 今日打卡状态（已打卡 ✓ / 未打卡）
3. 打卡入口：今日未打卡时显示「去打卡」橙色按钮；已打卡显示已完成态
4. 排行榜入口
5. 若为发起人：显示「结束 PK」按钮（二次确认）

**URL 参数**：`pkId: string`，可选 `fromInvite: '1'`（邀请进入时显示「加入确认弹窗」）

---

### 3.5 每日打卡 `subpackages/pk-core/pages/checkin/index`

**功能点**
1. 体重输入（数字键盘，精度 0.1kg）
2. 拍照上传（`wx.chooseMedia`，sourceType 限 camera，不允许相册）
3. 水印合成（Canvas 2D：日期 + 昵称 + 小程序名）
4. 可选：运动记录（类型 + 时长）
5. 提交打卡（防重复点击，提交中禁用按钮）
6. 成功后展示 Lottie 动效 + 积分变化

**重复打卡保护**
- 进入页面先查询今日打卡状态，若已打卡直接展示「今日已打卡」页面
- 后端返回 `ALREADY_CHECKED_IN` 错误码时，前端展示友好提示而非报错

---

### 3.6 排行榜 `subpackages/pk-core/pages/leaderboard/index`

**功能点**
1. 当前 PK 积分排行（实时）
2. 历史周榜（tab 切换）
3. 当前用户高亮
4. 每条目展示：排名 + 头像 + 昵称 + 积分 + 完成率环形进度

**URL 参数**：`pkId: string`

---

### 3.7 结算页 `subpackages/settlement/pages/index/index`

**功能点**
1. 结算结果：排名 + 积分 + 减重数据
2. 战报海报生成（Canvas 2D，含 PK 名称、名次、减重值、二维码）
3. 保存图片到相册（`wx.saveImageToPhotosAlbum`，需授权）
4. 分享给好友（`onShareAppMessage`）

**注意**：结算数据来自后端，前端只负责展示 + 海报生成，不做本地计算。

## 五、全局错误路由

| 场景 | 处理方式 |
|------|----------|
| token 过期（401） | 清空本地 token，`reLaunch` 到登录页 |
| 网络断开 | toast 提示「网络异常，请检查网络」，不跳页面 |
| 接口 500 | toast 提示「服务繁忙，请稍后重试」 |
| 页面不存在 | 微信默认 404 页面 |
| PK 不存在（pkId无效） | 在 pk-detail 的 onLoad 中处理，toast 后 navigateBack |
