## 二、页面结构（完整流程图）

```
小程序入口
    │
    ▼
pages/login/index（登录/授权）
    │ 登录成功
    ▼
pages/index/index（首页 TabBar）
    │
    ├── 【发起 PK】按钮
    │       └──▶ packagePK/pages/create/index（发起 PK）
    │                   │ 生成邀请
    │                   │ 分享到微信好友
    │                   └──▶ packagePK/pages/ranking/index（PK 详情）
    │
    ├── 点击【进行中 PK 卡片】
    │       └──▶ packagePK/pages/ranking/index（PK 详情）
    │               ├── Tab: 排行榜
    │               ├── Tab: 今日打卡入口
    │               │       └──▶ packagePK/pages/checkin/index（打卡）
    │               └── Tab: 个人详情（体重曲线/打卡日历）
    │
    ├── 点击【已结束 PK 卡片】
    │       └──▶ packageResult/pages/settle/index（结算）
    │                   └──▶ packageResult/pages/poster/index（海报）
    │
    └── 通过微信分享链接/小程序码进入
            └──▶ packagePK/pages/join/index（加入 PK）
                        │ 确认 + 录初始体重
                        └──▶ packagePK/pages/ranking/index（PK 详情）
```

## 四、路由使用规范

| 场景 | API | 说明 |
|------|-----|------|
| 普通页面跳转 | `wx.navigateTo` | 可返回上一页，用于大多数场景 |
| 打卡完成后跳排行榜 | `wx.redirectTo` | 不保留打卡页，避免返回重复打卡 |
| 切换 TabBar | `wx.switchTab` | TabBar 页之间切换 |
| 登录成功后进首页 | `wx.reLaunch` | 清空页面栈，避免返回到登录页 |
| 结算后分享 | `wx.navigateTo` 到 poster 页 | 保留结算页，可返回 |

## 五、页面详细设计

### 5.1 登录页（pages/login/index）

**进入条件**：本地无 token，或 token 过期

**交互流程**：
1. 展示 App Logo + 产品名 + Slogan
2. 展示《用户协议》和《隐私政策》链接（必须展示才能显示登录按钮）
3. 用户点击「微信一键登录」按钮
4. 调用 `wx.login` 获取 code → POST `/api/auth/login` → 拿到 JWT → 存 StorageSync
5. **首次登录**：弹出「敏感个人信息处理说明」独立同意弹框
   - 必须显式点击「我知道了，同意」才能继续
   - 拒绝则：toast 提示「需要您同意才能使用体重打卡功能」，可仅查看已有结算结果
6. 同意后 `wx.reLaunch` 到首页

**异常处理**：
- `wx.login` 失败 → toast「网络异常，请重试」
- 后端接口失败 → toast 错误信息，重试按钮

---

### 5.2 首页 / PK 列表（pages/index/index）

**页面结构**：
```
顶部：「减肥 PK」Logo + 「发起 PK」按钮（右上角）
内容区：
  ├── 进行中的 PK（卡片列表）
  │     每张卡片：PK名 + 进度条 + 剩余天数 + 我的今日打卡状态 + 我的排名
  └── 历史 PK（可折叠，展示已结束）
```

**数据**：`GET /api/pk/list` 返回用户参与的所有 PK

**空态**：无 PK 时展示「暂无 PK，立即发起一场！」引导按钮

---

### 5.3 发起 PK（packagePK/pages/create/index）

**表单字段**：

| 字段 | 类型 | 校验 |
|------|------|------|
| PK 名称 | 文本输入 | 2-20 字，必填 |
| PK 周期 | Picker 选择 | 7 / 14 / 21 / 30 / 60 / 90 天（自定义 7-90） |
| 开始日期 | DatePicker | 今天或未来 7 天内 |
| 积分规则说明 | 只读展示 | 告知用户计分规则 |

**提交**：`POST /api/pk/create` → 成功后展示邀请页（二维码 + 分享按钮）

---

### 5.4 加入 PK（packagePK/pages/join/index）

**进入方式**：
- 微信分享链接（场景值 1044/1007）
- 扫小程序码（场景值 1011）

**页面逻辑**：
1. 从 query 参数读取 `pkId`
2. `GET /api/pk/{pkId}` 获取房间信息
3. 展示：PK 名称 + 发起人 + 周期 + 当前人数 + 规则说明
4. 用户点击「确认加入」→ 弹出体重录入面板（WeightInput 组件）
5. 录入体重 → 拍照（CheckinCamera 组件）→ `POST /api/pk/join`

**防重加入**：已加入过的用户直接跳转 ranking 页

---

### 5.5 每日打卡（packagePK/pages/checkin/index）

**打卡状态机**：

```
今日未打卡 → 输入体重 + 拍照 → 选择饮食/运动状态 → 提交 → 打卡成功（lottie 动效）
今日已打卡 → 展示「今日已打卡」+ 已提交数据（只读）
PK 未开始 → 展示「PK 尚未开始」提示
PK 已结束 → 展示「PK 已结束」+ 结算入口
```

**体重打卡**（DR-006 待确认：是否允许当日覆盖）：
- 当前按「允许覆盖，取最后一次」实现
- 已打卡但未到截止时间：显示「修改今日体重」入口

---

### 5.6 排行榜 / PK 详情（packagePK/pages/ranking/index）

**Tab 设计**：
- Tab 1「排行榜」：综合积分榜（ScoreRankItem 列表）
- Tab 2「打卡」：今日打卡快捷入口 + 打卡日历（S 级功能，视时间决定）
- Tab 3「我的」：个人体重曲线（ECharts 折线图）+ 各维度积分明细

**排行榜条目**：
- 排名（1-N）
- 头像 + 昵称（脱敏）
- 综合积分（保留 1 位小数）
- 减重百分比（如 -5.2%）
- 连续打卡天数
- **不展示绝对体重**（隐私保护）

**数据更新**：T+1 00:05，展示「最后更新：昨日 00:05」

---

### 5.7 结算页（packageResult/pages/settle/index）

**内容**：
- 最终排名（前三名高亮）
- 各成员成绩卡：头像 + 昵称 + 初始体重 → 结束体重 + 减重量 + 减重% + 打卡率 + 最终名次
- 「生成战报海报」按钮 → 跳 poster 页

---

### 5.8 战报海报（packageResult/pages/poster/index）

**海报内容（Canvas 2D 绘制）**：
- 背景：品牌色渐变（橙 → 深橙）
- 左上角：App Logo + 「减肥 PK」文字
- 中间：PK 名称 + 周期（如 2026-07-14 ~ 2026-08-01）
- 个人数据区域：
  - 起始体重 → 结束体重（以箭头连接）
  - 减重量（kg）+ 减重%
  - 最终名次（如 🥇 第 1 名）
- 右下角：小程序码（`wx.getUnlimitedQRCode`）
- 底部：「扫码加入我的下一场 PK」

**操作**：「保存到相册」（`wx.saveImageToPhotosAlbum`）+ 「分享给好友」（转发图片）
